1) Big idea: What connect() does üß†

Before React Hooks existed, React-Redux had no useSelector / useDispatch. So to read state (and later to dispatch actions), we used the Connect API.

‚úÖ Mental model

connect(...)(YourComponent) creates a new component that is "wired" to Redux. That new component passes Redux state (and dispatch helpers) into your component as props.

In modern code, you typically do this: 

// Modern (hooks)
const balance = useSelector((state) => state.account.balance);
const dispatch = useDispatch();

In older code, you do this: 

// Legacy (connect)
export default connect(mapStateToProps)(BalanceDisplay);

2) mapStateToProps: mapping store ‚Üí props üó∫Ô∏è

The most common first step is reading state. With connect, you define a function called mapStateToProps.

‚úÖ What mapStateToProps receives
  • It receives the entire Redux store state (often named state).
  • You return an object that becomes props in your component.
// Example: take only what you need from the store
function mapStateToProps(state) {
  return {
    balance: state.account.balance,
  };
}

That means your component will receive a prop called balance. So inside the component, you can use it like a normal React prop.

3) How the wrapping works (HOC pattern) üîÅ

The old API uses a pattern called a Higher-Order Component (HOC): a function that takes a component and returns a new component. 

// Shape (conceptually)
connect(mapStateToProps)            // returns a function
  (BalanceDisplay)                  // that function returns a NEW component
üß† Why it feels "messy"

It‚Äôs function calls returning functions, instead of direct hook calls inside the component. This is one big reason hooks became the preferred approach.

4) Example: BalanceDisplay using connect üí∞

Goal: show the current balance from state.account.balance, formatted using a helper like formatCurrency.

Step-by-step ‚úÖ

  1. Create mapStateToProps to select the balance
  2. Use the injected prop inside the component
  3. Export the connected component
import { connect } from "react-redux";

function BalanceDisplay({ balance }) {
  return (
    <p className="balance">
      {formatCurrency(balance)}
    </p>
  );
}

function mapStateToProps(state) {
  return {
    balance: state.account.balance,
  };
}

export default connect(mapStateToProps)(BalanceDisplay);
‚úÖ What you gain
  • Your component automatically re-renders when the mapped state changes (e.g., deposit/withdraw).
  • Your component code stays ‚Äúpure React props‚Äù instead of directly reading the store.

üßæ Why you still learn this

‚úÖ Good selector hygiene

üßæ Key definitions (beginner-friendly)

1) React Component (Blueprint / Template)

What you write.

  • A component is usually a JavaScript function (or class) that describes a piece of UI.
  • It returns React elements (often written in JSX).
  • Think: ‚Äúrecipe‚Äù or ‚Äúblueprint‚Äù ‚Äî not the actual thing on screen yet.

2) Component Instance (A Living Copy in the Tree)

What React creates when you use the component.

  • Each time you write <MyComponent /> in JSX, React creates an instance.
  • React calls the component function for each instance (e.g., 3 usages ‚Üí 3 calls).
  • Each instance owns its own: props, state, and lifecycle.

3) React Element (Immutable Object in Memory)

What JSX becomes.

  • A React element is the result of calling React.createElement(...).
  • It‚Äôs a plain JS object that describes what should appear (type + props + children).
  • It is immutable (React creates new elements instead of mutating old ones).

4) DOM Element (Real Browser Node)

What the user actually sees.

  • DOM elements are the real HTML nodes in the browser (e.g., <div>, <button>).
  • React elements are not DOM nodes; they are just descriptions.
  • React uses the element description to update the DOM efficiently.

Example A: One component, multiple instances

  return (
    <button
      className={num === activeTab ? "active" : ""}

üíº Interview-style Q&A

Q1: What is a React element?

Answer (safe and strong):

A React element is an immutable JavaScript object created by JSX / React.createElement. It describes what should appear in the UI and is later used by React to update the DOM.

Q2: Component vs instance?

A component is the blueprint (function/class). An instance is a specific usage of that component in the component tree, with its own props, state, and lifecycle.

Q3: Are React elements the same as DOM elements?

No. React elements are in-memory descriptions (JS objects). DOM elements are actual browser nodes rendered on the page.

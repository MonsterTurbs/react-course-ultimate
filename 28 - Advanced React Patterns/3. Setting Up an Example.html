Why ‚Äúcustom hooks only‚Äù is not enough

You could extract the list logic (collapsed state, show-less state, displayItems calculation) into a custom hook. That helps reuse stateful logic.

But you would still have to duplicate the UI structure in two separate components (ProductList + CompanyList): buttons, layout, <ul className=...>, and conditional rendering.

In short: custom hooks reuse logic, but not the JSX layout that you also want to share.

Why the children prop doesn‚Äôt solve it (in this case)

A natural idea is: ‚ÄúWhat if we pass the <ul> as children?‚Äù But then you hit a key limitation:

  • The list component controls displayItems (sometimes all items, sometimes first 3). If the caller owns the <ul>, the caller needs access to displayItems.
  • You would likely end up duplicating the <ul className=...> and the mapping logic in two places anyway.

Key takeaway: children is great for passing content. Here you need to pass instructions (a recipe) for how to render each item.

Preview: what ‚Äúrender props‚Äù will provide

Render props means: instead of passing JSX as children, you pass a function. The List component can call that function for each item:

  items={products}

  title="Companies"
  items={companies}

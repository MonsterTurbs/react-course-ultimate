1) ‚ÜîÔ∏è/‚ÜïÔ∏è Build a reusable Row component

The UI often needs sections placed side-by-side (horizontal) or stacked (vertical). Instead of repeating flex CSS, create a reusable component.

üìÑ File: ui/Row.jsx

import styled, { css } from "styled-components";

const Row = styled.div`
  display: flex;

  ${props =>
    props.type === "horizontal" &&
    css`
      justify-content: space-between;
      align-items: center;
    `}

  ${props =>
    props.type === "vertical" &&
    css`
      flex-direction: column;
      gap: 1.6rem;
    `}
`;

Row.defaultProps = {
  type: "vertical",
};

export default Row;
Default props: If you don‚Äôt pass type, it becomes "vertical" automatically.

Usage examples

import Heading from "./ui/Heading";

  return (
    <Row type="vertical">

The inner <Row> defaults to type="vertical" because of Row.defaultProps.

The transcript mentions you can also set defaults via destructuring like ({ type = "vertical" }). But for styled-components (where there‚Äôs no component function body), defaultProps is convenient.

2) üîò Upgrade Button: sizes + variations

Instead of hardcoding one look, we want a flexible API:

  • üìè size: small | medium | large
  • üé® variation: primary | secondary | danger
This creates 3 √ó 3 = 9 possible button combinations with one component.

Key technique: store CSS snippets in objects

The starter code already includes objects like sizes and variations. Your job is to select the correct entry based on props: 

${props => sizes[props.size]}
${props => variations[props.variation]}

üìÑ File: ui/Button.jsx (core idea)

import styled from "styled-components";

const sizes = {
  small: "font-size: 1.2rem; padding: 0.4rem 0.8rem;",
  medium: "font-size: 1.4rem; padding: 0.8rem 1.2rem;",
  large: "font-size: 1.6rem; padding: 1.2rem 1.6rem;",
};

const variations = {
  primary: "background-color: var(--color-brand-600); color: var(--color-brand-50);",
  secondary: "background-color: var(--color-grey-0); color: var(--color-grey-600); border: 1px solid var(--color-grey-200);",
  danger: "background-color: var(--color-red-700); color: var(--color-red-50);",
};

const Button = styled.button`
  border: none;
  border-radius: var(--border-radius-sm);
  box-shadow: var(--shadow-sm);
  cursor: pointer;

  ${props => sizes[props.size]}
  ${props => variations[props.variation]}
`;

Button.defaultProps = {
  variation: "primary",
  size: "medium",
};

export default Button;
Your actual project may already have richer CSS inside sizes and variations. The important concept is the lookup by prop.

Usage examples

import Button from "./ui/Button";

function ButtonsDemo() {
  return (
    <>
      <Button>Primary (default)</Button>
      <Button variation="secondary" size="small">Secondary Small</Button>
      <Button variation="danger" size="large">Danger Large</Button>
    </>
  );
}

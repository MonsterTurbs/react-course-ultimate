1) What is a Root Layout? üåç

In the Next.js App Router, every app needs a global layout called the Root Layout. This layout wraps the entire application, meaning it applies to every route.

Why Next.js re-created layout.js earlier:

When you deleted layout.js, Next.js complained that there was no root layout and automatically generated one. That‚Äôs because the root layout is required.

2) Root Layout MUST include <html> and <body> ‚úÖ

The root layout defines the global HTML structure. That‚Äôs why it must return <html> and <body>. You can also set attributes like lang.

app/layout.js (minimal structure)
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        {children}
      </body>
    </html>
  );
}
Why no <head> here?

In App Router, you typically control head info (like title) using conventions such as exporting metadata (shown below).

3) Why did page content disappear? ü§î

When you first added only the Navigation inside the root layout, all pages showed the navigation, but none of the page-specific content appeared.

Cause:

The layout was returning only the navigation. Since every page is wrapped by this layout, the page output had no place to render.

The fix: use the children prop üë∂

Layouts receive a children prop. Next.js automatically fills children with the current page (or nested layout) for the route.

Root Layout with navigation + page content
import Navigation from "./components/navigation";

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <Navigation />
        <main>{children}</main>
      </body>
    </html>
  );
}
React connection:

children is a core React concept (also similar to React Router‚Äôs <Outlet /> idea). That‚Äôs why knowing React well makes Next.js much easier.

4) Shared UI: Navigation + Footer (renders on all pages) üß©

Anything you put in the layout outside of {children} becomes shared UI. Example: a footer.

Shared footer in the layout
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <Navigation />
        <main>{children}</main>
        <footer>Copyright by The Wild Oasis</footer>
      </body>
    </html>
  );
}
Benefit:

You no longer need to import Navigation on every page. The layout gives you consistent UI across the entire app.

Performance note (from the lecture):

Shared layout UI typically does not re-render the same way as full-page reloads when navigating. It helps keep navigation smooth.

5) Page Title + Metadata using conventions üè∑Ô∏è

Instead of writing a manual <title> tag, you can export a constant named metadata from your layout. Next.js then generates the correct head elements.

Export metadata from the layout
export const metadata = {
  title: "The Wild Oasis",
};
Key idea:

Next.js is heavily convention-based: file names like page.js, layout.js, and exports like metadata unlock framework features without extra boilerplate.

7) Mini Quiz (self-check) üß™

Q1) Why is the root layout required in every Next.js App Router app?
Because it wraps the entire application and defines the global HTML structure. Next.js needs it to render pages correctly.
Q2) What two tags are mandatory in the root layout?
The <html> and <body> tags.
Q3) What does the children prop represent in a layout?
It represents the content of the currently active page (or nested layout) that Next.js injects automatically.
Q4) How do you set the page title using a Next.js convention?
Export metadata from the layout (e.g., export const metadata = { title: "..." }).
Q5) Where do you put static assets like icons?
Inside the public/ folder.

üéØ Why we need a loading indicator

When you navigate (e.g., Home ‚Üí Cart ‚Üí Menu), there can be a small delay between the click and the data arriving. During that delay, the app should show feedback (spinner/loader) so users know something is happening.

‚úÖ The goal: show a loader while route data is being fetched.

üß† The problem: ‚ÄúWhere is the isLoading state?‚Äù

Unlike a typical useEffect fetch, there is no local component isLoading state. With React Router loaders, the router triggers fetching, so we should read the router‚Äôs own navigation state.

üß≠ Solution: use useNavigation()

React Router provides useNavigation() (not useNavigate). This hook exposes the current navigation status of the entire router.

State Meaning When you might see it
idle Nothing is happening right now Normal browsing after data has loaded
loading A route transition is loading data Navigating to a page that has a loader
submitting A route action is submitting form data Submitting a form tied to an action
‚úÖ This state is global: if any page is loading, navigation.state reflects it.

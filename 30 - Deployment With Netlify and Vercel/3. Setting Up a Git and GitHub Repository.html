üß† Beginner-friendly concepts

1) What is Git?

Git is a version control system. Think of it as ‚Äúsave points‚Äù for your code. Each save point is called a commit (üì∏ snapshot).

2) What is a repository?

A repository (repo) is a special folder that tracks file changes over time. Git creates a hidden folder called .git that stores the history.

3) Local repo vs Remote repo

  • üíª Local repo: on your computer
  • ‚òÅÔ∏è Remote repo: online (GitHub / GitLab)
‚úÖ Key idea

Local repo = where you code. Remote repo = backup + sharing + deployment pipelines (CI/CD).

üß© Step-by-step workflow (the ‚Äúhappy path‚Äù)

A) Install Git

  1. üåê Download Git for your OS (Windows/macOS/Linux)
  2. üß™ Confirm it works in your terminal:
git --version

B) Initialize your project as a Git repository

git init

üìù This creates a .git folder that stores the change history.

C) Check current status

git status

Git will show which files are:

  • üÜï Untracked: not yet added to Git
  • ‚úèÔ∏è Modified: changed since last commit
  • ‚úÖ Staged: ready to be committed

D) Create a README.md (recommended)

üìÑ README.md template 
# The Wild Oasis

Main project in Jonas Schmedtmann's Ultimate React Course.

Built with React Query and Supabase.

Markdown basics: # = H1, ## = H2, etc.

E) Stage files (add to staging area)

Staging area = the list of changes you want included in the next commit.

git add -A

F) Commit (save a snapshot)

git commit -m "initial commit"
üìù Commit message tip

Write messages like commands: ‚Äúadd login form‚Äù, ‚Äúfix bug‚Äù, ‚Äúupdate README‚Äù.

‚òÅÔ∏è GitHub setup (remote repository)

A) Create a GitHub account + verify email

  • üë§ Sign up / sign in to GitHub
  • üìß Verify your email (important for pushing and settings)

B) Create a new repository

  • ‚ûï Click the ‚Äúplus‚Äù ‚Üí New repository
  • üìõ Name: e.g. The Wild Oasis
  • üîí Public or Private (your choice)
  • ‚ö†Ô∏è Do NOT initialize with README/.gitignore/license (because you already have a local repo)
Why avoid GitHub auto-README?

If both local and remote create initial commits separately, you can get ‚Äúhistory mismatch‚Äù and need merges. Starting with an empty remote is simpler for beginners.

üîê Personal Access Token (PAT)

GitHub now often requires a token instead of your password when pushing via HTTPS. A PAT is basically a password substitute.

  1. Go to GitHub Settings (avatar menu)
  2. Scroll down ‚Üí Developer settings
  3. Personal access tokens ‚Üí generate new token
  4. Set expiration (example: 30 days)
  5. Select scope: ‚úÖ repo (needed for pushing to repos)
  6. Generate token ‚Üí copy it and store safely
üö® Treat your token like a password
  • Do not paste it into public places or screenshots
  • If leaked, revoke/delete it immediately
  • You may not be able to view it again after closing the page

üîó Connect local repo to GitHub (remote)

1) Add the remote called origin

Copy your GitHub repo HTTPS URL, then:

git remote add origin https://github.com/<your-username>/<your-repo>.git

2) Push your code to GitHub

git push -u origin main
‚úÖ What ‚Äú-u‚Äù means

It sets an upstream tracking branch, so later you can just run git push without extra parameters.

3) When Git asks for credentials

  • Username/email: your GitHub email/username
  • Password: your Personal Access Token (PAT)

üß™ Mini example: modify ‚Üí add ‚Üí commit ‚Üí push

  1. Edit a file (example: README.md)
  2. Check changes:
git status

Stage changes:

git add -A

Commit:

git commit -m "update README"

Push:

git push
üìå Mental model

Working directory ‚Üí (git add) ‚Üí Staging area ‚Üí (git commit) ‚Üí Local history ‚Üí (git push) ‚Üí GitHub

üìÅ Why node_modules and dist are not on GitHub

You may notice GitHub shows only source and public folders (and configs) but not node_modules or dist.

Folder/File Should it be in Git? Reason
node_modules/ No ‚ùå Huge; can be reinstalled anytime using npm install
dist/ Usually No ‚ùå Build output; can be recreated using npm run build
src/, public/ Yes ‚úÖ Your actual source code and static assets
.gitignore Yes ‚úÖ Tells Git what to ignore (node_modules, dist, env files, etc.)
‚úÖ Best practice

Commit source code, not generated output. Your build server (Netlify/Vercel) will build the project from source.

üßæ Quick ‚ÄúCommand Cheat Sheet‚Äù

  • üìç Initialize repo: git init
  • üîé Check status: git status
  • ‚ûï Stage all files: git add -A
  • üì∏ Commit: git commit -m "message"
  • üîó Add remote: git remote add origin <url>
  • üöÄ Push first time: git push -u origin main
  • üöÄ Push later: git push

üß† Common mistakes (and fixes)

1) ‚ÄúAuthentication failed‚Äù
  • Use your Personal Access Token as the password (not your GitHub password)
  • Token must have repo scope
  • If expired, generate a new token
2) Branch name mismatch (master vs main)

Your default branch might be master on older setups. Check with:

git status

Then push using the correct branch name.

3) Forgot to stage files

If commit says ‚Äúnothing to commit‚Äù, run:

git add -A

1) What we are building

‚≠ê Key takeaway:

The items state lives in App, so the ‚Äúclear everything‚Äù logic belongs in App too. The button is in PackingList, so we pass a function down as a prop.

2) Step 1 ‚Äî Add the button in PackingList

Inside PackingList, you already have the sorting dropdown. Right after the <select>, add: 

// PackingList.jsx (inside <div className="actions">)
<button onClick={onClearList}>Clear list</button>
‚úÖ Why no wrapper function?

Unlike delete/toggle (which need an id), clearing needs no arguments, so you can pass the handler directly.

3) Step 2 ‚Äî Create the handler in App

In App, define a new function near the other handlers (handleAddItems, handleDeleteItem, handleToggleItem). 

// App.jsx
function handleClearList() {
  setItems([]); // reset to initial value
}
üß† Mental model:

Clearing the list is just setting state back to its initial value: an empty array.

4) Step 3 ‚Äî Pass the handler down via props

Since the button is inside PackingList, pass the handler from App down as a prop. 

// App.jsx
<PackingList
  items={items}
  onDeleteItem={handleDeleteItem}
  onToggleItem={handleToggleItem}
  onClearList={handleClearList}
/>

Then receive it in PackingList:

// PackingList.jsx
function PackingList({ items, onDeleteItem, onToggleItem, onClearList }) {
  // ...
}
üîÅ This is child-to-parent communication (inverse data flow)

The click happens in a child (PackingList), but it updates state in the parent (App) using a function prop.

5) Add confirmation to prevent accidents üõ°Ô∏è

In the demo, users must confirm before wiping everything. Use window.confirm(). 

// App.jsx
function handleClearList() {
  const confirmed = window.confirm("Are you sure you want to delete all items?");
  if (confirmed) setItems([]);
}
How window.confirm works:
  • ‚úÖ Clicking OK returns true
  • ‚ùå Clicking Cancel returns false

6) Common mistakes (and fixes) üßØ

‚ùå Putting clear logic in PackingList

PackingList does not own the items state, so it should not call setItems directly.

‚úÖ Fix: keep setItems([]) in App, pass a handler down.

‚ùå Writing onClick={onClearList()}

That calls the function immediately on render.

‚úÖ Fix: use onClick={onClearList}.

‚ùå Forgetting to add/receive the prop

If the button does nothing, verify you passed onClearList into PackingList and destructured it.

1) Context: Why useFormStatus is not enough here

In the Update Profile form, a loading label like ‚ÄúUpdating‚Ä¶‚Äù was easy because the Server Action was used as the form‚Äôs action. That setup enables the useFormStatus hook.

‚ö†Ô∏è Key constraint

useFormStatus only works for components rendered inside a <form> that is submitting. If your Server Action is called from a button click (no form submit), useFormStatus cannot detect that.

Example situation: ‚ÄúDelete reservation‚Äù is triggered by clicking a delete button. No form submit is happening, so we need another mechanism.

2) The solution: useTransition

useTransition is a React 18 hook (a ‚Äúconcurrent feature‚Äù). In plain React, it is used to mark a state update as a transition so it can run in the background without blocking the UI.

üí° Next.js twist

In Next.js, we can also treat a Server Action invocation like a transition. That gives us a boolean flag (pending) to drive a loading indicator while the action is running.

What useTransition returns

  • isPending (boolean) ‚Üí true while the transition is running
  • startTransition(fn) ‚Üí wrapper that marks fn as the transition work

3) Practical recipe: Delete Reservation + Spinner

Goal: When user clicks Delete, show a spinner while the Server Action deletes the booking and the UI is being updated.

‚úÖ Pattern
  • Move the delete logic into handleDelete
  • Wrap the server-action call inside startTransition(() => ...)
  • Render spinner when isPending === true

Example code (Client Component)

'use client';

import { useTransition } from 'react';
import SpinnerMini from './SpinnerMini';
import { deleteReservation } from '@/app/_lib/actions'; // your Server Action

export default function DeleteReservation({ bookingId }) {
  const [isPending, startTransition] = useTransition();

  function handleDelete() {
    // Optional safety prompt so users don't delete by accident
    const ok = confirm('Are you sure you want to delete this reservation?');
    if (!ok) return;

    startTransition(() => {
      // Server Action call (runs on the server)
      deleteReservation(bookingId);
    });
  }

  return (
    <button
      type="button"
      onClick={handleDelete}
      disabled={isPending}
      className="text-sm underline disabled:opacity-50"
      aria-busy={isPending}
    >
      {!isPending ? (
        <>üóëÔ∏è Delete</>
      ) : (
        <span className="mx-auto">
          <SpinnerMini />
        </span>
      )}
    </button>
  );
}
‚úÖ What the user experiences
  • Click Delete ‚Üí spinner shows immediately
  • Server Action runs (delete in DB + revalidatePath)
  • Updated UI renders: the reservation disappears

4) What‚Äôs happening behind the scenes

  • Next.js executes the Server Action on the server (as if it were an API endpoint).
  • The UI is refreshed by re-fetching data (cache revalidation), not by client state mutation.
  • Next.js wraps many navigation and data updates in transitions; here we are doing it explicitly.
‚ö†Ô∏è Common misconception

‚ÄúDo we need React state to remove the deleted item from the list?‚Äù
No. In Next.js Server Actions patterns, the usual approach is: mutate data ‚Üí revalidate cache ‚Üí page re-renders with fresh data.

5) When to use which loading hook?

üßæ Use useFormStatus when‚Ä¶
  • Server Action is connected to a <form action={...}>
  • You want pending state for form submit (e.g., ‚ÄúUpdating‚Ä¶‚Äù, disable button)
üßµ Use useTransition when‚Ä¶
  • Server Action is called from a button click / event handler (not a form submit)
  • You want UI to stay responsive + get isPending for a spinner/disabled state

6) Quick checklist

  • ‚úÖ Component is a Client Component ('use client')
  • ‚úÖ Import useTransition from react
  • ‚úÖ Wrap the action call with startTransition(() => ...)
  • ‚úÖ Use isPending to render spinner + disable button
  • ‚úÖ Server Action handles DB mutation + revalidatePath so UI updates correctly

Summary

When your Server Action is triggered by a form submission, useFormStatus is the clean solution for pending UI. When the action is triggered by a button click (no form submit), use useTransition: wrap the call with startTransition and use isPending for a spinner/disabled state.

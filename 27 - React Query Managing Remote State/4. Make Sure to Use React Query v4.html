Big Idea (What the lecturer wants you to do)

React Query recently released v5, but the course project is built using v4. To avoid mismatches (errors or different APIs), you should install React Query v4 exactly as shown in the course.

‚ö†Ô∏è Course compatibility rule:
Follow the project steps with React Query v4. Using v5 is possible, but you must rename a couple of APIs.

Install React Query v4

Use this exact install command (note the @4):

npm i @tanstack/react-query@4

If you decide to use v5 anyway

The lecturer says the changes needed are small. Here are the two renames you must apply.

In v4 In v5 Where you‚Äôll see it
isLoading isPending Query/mutation result flags (e.g., useQuery, useMutation)

Example: isLoading ‚Üí isPending

If your code checks loading state like this in v4, update it for v5: 

// v4
const { data, isLoading, error } = useQuery({
  queryKey: ["cabins"],
  queryFn: getCabins,
});

if (isLoading) return <Spinner />;
// v5 (rename)
const { data, isPending, error } = useQuery({
  queryKey: ["cabins"],
  queryFn: getCabins,
});

if (isPending) return <Spinner />;

Example: cacheTime ‚Üí gcTime

If you configure cache cleanup time, rename the option in v5:

  defaultOptions: {
    queries: {
      staleTime: 60 * 1000,
      cacheTime: 5 * 60 * 1000,
    },
  },
});
// v5 (rename)

  defaultOptions: {
    queries: {
      staleTime: 60 * 1000,
      gcTime: 5 * 60 * 1000,
    },
  },

‚úÖ Common patterns you should memorize

‚ö†Ô∏è Troubleshooting and gotchas

1) ‚ÄúMy focus width breaks the header on mobile‚Äù

Fix by scoping the focus width change to larger screens: sm:focus:w-72 (instead of focus:w-72).

2) ‚ÄúMy placeholder styles don‚Äôt show‚Äù

Placeholder classes only affect the placeholder text. Ensure your input actually has a placeholder attribute.

3) ‚ÄúI removed outline, now focus is invisible‚Äù

Always pair focus:outline-none with a replacement, typically a ring: focus:ring + a ring color.

üìå Mini checklist (apply to any input)

  • Is it readable? (text size + color)
  • Is it clickable/tappable? (padding + height)
  • Does it look distinct from the background? (border or background)
  • Does it have a good focus state? (ring, not just outline removal)
  • Does it behave well on mobile? (responsive widths; avoid layout-breaking focus effects)

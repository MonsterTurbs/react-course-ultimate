üéØ Why optional chaining exists

Real-world data (APIs, JSON, database results) is often incomplete. Sometimes a nested object is missing. If you access deep properties without checking, JavaScript can throw runtime errors.

Goal ‚úÖ

Safely read nested properties without writing many if checks.

üí• The common error

If a value is undefined (or null), and you try to access a property on it, you get an error like: Cannot read properties of undefined

Example data shape (conceptual)

// book.reviews can contain:
// - reviews.goodreads
// - reviews.librarything (might be missing!)

// Example path we want:
book.reviews.librarything.reviewsCount

What goes wrong

// If book.reviews.librarything is undefined:
// Then this becomes: undefined.reviewsCount  ‚ùå
const count = book.reviews.librarything.reviewsCount;

Key point ‚ö†Ô∏è

The crash happens before your app can do anything else (especially painful in React UIs).

üß© How ?. works

Optional chaining lets you say: ‚ÄúOnly continue this chain if the value before it exists.‚Äù

Normal chaining

book.reviews.librarything.reviewsCount

Optional chaining

book.reviews.librarything?.reviewsCount
// If librarything is undefined/null ‚Üí result becomes undefined (no crash)

In other words: if book.reviews.librarything is missing, JavaScript stops and returns undefined.

You can chain multiple times (extra safe)

book.reviews?.librarything?.reviewsCount
// Safe even if reviews is missing too

üßØ Pair it with ?? for safe defaults

Optional chaining prevents crashes, but it can still return undefined. If you do math with undefined, you might get NaN (Not a Number).

Problem: number + undefined ‚Üí NaN

const goodreads = 500;
const librarything = undefined;

console.log(goodreads + librarything);
// ‚Üí NaN

Fix: default missing values to 0 using ??

const librarythingCount = book.reviews?.librarything?.reviewsCount ?? 0;
// If missing ‚Üí 0
// If present ‚Üí actual number

Why ?? (not ||) ‚úÖ

?? only falls back for null/undefined. It keeps valid 0 values.

üß™ Example: getTotalReviewCount(book)

We want total reviews count = Goodreads + LibraryThing. But some books might not have LibraryThing data.

Naive version (can crash) ‚ùå

function getTotalReviewCount(book) {
  const goodreads = book.reviews.goodreads.reviewsCount;
  const librarything = book.reviews.librarything.reviewsCount; // can crash

  return goodreads + librarything;
}

Safe version (recommended) ‚úÖ

function getTotalReviewCount(book) {
  const goodreads = book.reviews?.goodreads?.reviewsCount ?? 0;
  const librarything = book.reviews?.librarything?.reviewsCount ?? 0;

  return goodreads + librarything;
}

// Works even when one side is missing

Why this works

  • üîó ?. prevents ‚ÄúCannot read properties of undefined‚Äù.
  • üßØ ?? 0 prevents NaN when doing math.

üß† Tips & pitfalls

  • Optional chaining returns undefined when it stops early. Plan a fallback if needed.
  • Use ?? for defaults (especially with numbers where 0 is valid data).
  • Be consistent: if you‚Äôre unsure about the data shape, use ?. at every risky hop (a?.b?.c).
  • Don‚Äôt overuse blindly: if something must exist, an error might reveal a real bug in your data.

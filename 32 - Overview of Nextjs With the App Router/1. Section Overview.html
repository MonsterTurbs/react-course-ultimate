Q1: What is the main difference between CSR and SSR?
CSR renders the initial UI in the browser after downloading JavaScript. SSR renders initial HTML on the server first, then hydrates in the browser for interactivity.
Q2: Why is hydration needed after SSR?
Because the initial HTML is not interactive by itself. Hydration connects React state and event handlers so UI controls work.
Q3: In one sentence, what is React Server Components?
A React architecture that renders some components on the server (for data/heavy work) while keeping interactive components on the client.
Q4: What should you focus on first when learning Next.js?
The server/client mental model: what runs where, why, and what is sent over the network.

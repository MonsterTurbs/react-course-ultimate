3) State plan

‚úÖ The 3 pieces of state you need

  • bill (string or number) ‚Äî value of the bill input
  • percentage1 (number) ‚Äî your satisfaction %
  • percentage2 (number) ‚Äî friend‚Äôs satisfaction %

‚¨ÜÔ∏è Where should state live?

Even if inputs are inside child components, you will need these values in the parent to compute the output. So the state should live in TipCalculator.

‚¨ÜÔ∏è Lifting state up means: move state to the closest common parent that needs the data.

4) Controlled inputs recipe (always the same)

  1. üß† Define state in the parent: const [bill, setBill] = useState("")
  2. üì¶ Pass value down as a prop: <BillInput bill={bill} />
  3. üîÅ Pass setter (or handler) down so child can update parent state
  4. üéØ Child uses value={bill} and onChange={...}
‚ö†Ô∏è Common bug: event.target.value is a string. If you need numbers, convert with Number(...).

5) Derived state (tip is NOT state)

üßÆ Tip can be computed from existing state (bill, percentage1, percentage2), so you should NOT store tip in its own useState. 

// Derived state (computed values)
const tip = bill * ((percentage1 + percentage2) / 2 / 100);
const total = bill + tip;
  • ‚úÖ No extra syncing work
  • ‚úÖ No extra renders from updating multiple related state variables
  • ‚úÖ Single source of truth: bill + percentages

6) children prop use-case (nice touch)

üß© You can reuse SelectPercentage twice and pass the label text as children. That means you can write: 

<SelectPercentage percentage={percentage1} onSelect={setPercentage1}>
  How did you like the service?
</SelectPercentage>

<SelectPercentage percentage={percentage2} onSelect={setPercentage2}>
  How did your friend like the service?
</SelectPercentage>

The content inside the tags becomes children inside the component.

7) Reset + conditional rendering

Reset handler

function handleReset() {
  setBill("");
  setPercentage1(0);
  setPercentage2(0);
}

Only show Output + Reset when it makes sense

{bill > 0 && (
  <>
    <Output bill={bill} tip={tip} />
    <Reset onReset={handleReset} />
  </>

8) Complete example solution (reference)

‚úÖ Use this as a reference after trying on your own. If your solution differs but works, that is fine. 
import { useState } from "react";

export default function TipCalculator() {
  const [bill, setBill] = useState("");
  const [percentage1, setPercentage1] = useState(0);
  const [percentage2, setPercentage2] = useState(0);

  const numericBill = Number(bill);
  const tip = numericBill * ((percentage1 + percentage2) / 2 / 100);

  function handleReset() {
    setBill("");
    setPercentage1(0);
    setPercentage2(0);
  }

  return (
    <div>
      <BillInput bill={bill} onSetBill={setBill} />

      <SelectPercentage percentage={percentage1} onSelect={setPercentage1}>
        How did you like the service?
      </SelectPercentage>

      <SelectPercentage percentage={percentage2} onSelect={setPercentage2}>
        How did your friend like the service?
      </SelectPercentage>

      {numericBill > 0 && (
        <>
          <Output bill={numericBill} tip={tip} />
          <Reset onReset={handleReset} />
        </>
      )}
    </div>
  );
}

function BillInput({ bill, onSetBill }) {
  return (
    <div>
      <label>How much was the bill? </label>
      <input
        type="text"
        placeholder="Bill value"
        value={bill}
        onChange={(e) => onSetBill(e.target.value)}
      />
    </div>
  );
}

function SelectPercentage({ percentage, onSelect, children }) {
  return (
    <div>
      <label>{children} </label>
      <select
        value={percentage}
        onChange={(e) => onSelect(Number(e.target.value))}
      >
        <option value={0}>Dissatisfied (0%)</option>
        <option value={5}>It was okay (5%)</option>
        <option value={10}>It was good (10%)</option>
        <option value={20}>Absolutely amazing (20%)</option>
      </select>
    </div>
  );
}

function Output({ bill, tip }) {
  const total = bill + tip;

  return (
    <h3>
      You pay ${total.toFixed(2)} (${bill.toFixed(2)} + ${tip.toFixed(2)} tip)
    </h3>
  );
}

function Reset({ onReset }) {
  return <button onClick={onReset}>Reset</button>;
}

9) Common beginner mistakes (so you can avoid them)

‚ö†Ô∏è Storing derived values as state

Don‚Äôt do const [tip, setTip] = useState(...) if it can be computed from existing state.

‚ö†Ô∏è Forgetting number conversion

e.target.value is a string. Use Number(...) when needed.

‚ö†Ô∏è Mutating props/state

Update state immutably; never directly change props.

‚ö†Ô∏è Calling handlers immediately

Use onClick={() => ...} when you need to pass arguments.

üß† Key Concepts (Beginner-friendly)

1) ‚ÄúBookings‚Äù vs ‚ÄúReservations‚Äù

In this project, records are stored in a bookings table. The UI calls them reservations because users are not paying online yet. The naming is a UX decision; technically, you still fetch from bookings.

2) Fetch the guest‚Äôs bookings using the session guestId

You previously stored guestId into the session (in Auth.js callbacks). Now you use it to fetch only the logged-in guest‚Äôs bookings.

üîê Why it matters: guestId is the ‚Äúforeign key‚Äù used to find bookings that belong to a user.

3) Server Component pattern

This page is typically a Server Component (default in App Router). That means you can:

  • Call await auth() directly (server-only).
  • Fetch data with await getBookings(...) without client state.
  • Return JSX that already contains the data.

üß© Implementation Recipe

Step A: Fetch bookings in the Reservations page

1) Make the page component async. 2) Read the session. 3) Use session.user.guestId as input to getBookings. 

// app/account/reservations/page.js (example)
import { auth } from "@/app/_lib/auth";
import { getBookings } from "@/app/_lib/data-service";
import ReservationCard from "@/app/_components/ReservationCard";

export default async function ReservationsPage() {
  const session = await auth();

  // If you want extra safety:
  // if (!session?.user?.guestId) throw new Error("Not authenticated");

  const bookings = await getBookings(session.user.guestId);

  if (!bookings.length)
    return (
      <div>
        <h2>Your reservations</h2>
        <p>You have no reservations yet. Start by selecting a cabin.</p>
      </div>
    );

  return (
    <div>
      <h2>Your reservations</h2>
      <ul>
        {bookings.map((booking) => (
          <li key={booking.id}>
            <ReservationCard booking={booking} />
          </li>
        ))}
      </ul>
    </div>
  );
}
üß∑ Tip: Your data service might name it getBookings or getBookingsByGuest. The important part is: it filters by guestId.

Step B: ReservationCard using Next.js <Image /> (Responsive technique)

Common pattern for responsive images in Next.js: put <Image fill /> inside a relative parent with a fixed height (and often a square aspect ratio).

üõ°Ô∏è Hide Edit/Delete for Past Reservations

Past reservations should not be editable or deletable. You can conditionally render the buttons based on whether the start date is in the past. 

// ReservationCard.jsx (snippet)
import { isPast } from "date-fns";
import Link from "next/link";

export default function ReservationCard({ booking }) {
  const startDate = new Date(booking.startDate);

  return (
    <div>
      {/* ...booking details... */}

      {!isPast(startDate) ? (
        <>
          <Link href={`/account/reservations/edit/${booking.id}`} className="btn">
            ‚úèÔ∏è Edit
          </Link>

          <button className="btn danger">üóëÔ∏è Delete</button>
        </>
      ) : null}
    </div>
  );
}
‚úÖ Pattern: condition ? <>...</> : null cleanly ‚Äúremoves‚Äù UI when not allowed.

Also prefer <Link /> over raw <a href="..."> in Next.js for client-side navigation.

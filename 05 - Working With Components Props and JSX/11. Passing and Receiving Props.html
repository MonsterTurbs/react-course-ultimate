üß† What are props?

Props (short for properties) are how we pass data between components ‚Äî specifically from a parent component to a child component.

Helpful mental model üì°

Props are like a communication channel from parent ‚Üí child.

üçï Why props are needed for reusability

You already reused the Pizza component multiple times, but all pizzas looked the same. That‚Äôs because the component had hard-coded data.

Before props ‚ùå

Reusing <Pizza /> creates copies of the same UI with the same data.

After props ‚úÖ

Each <Pizza /> can receive different data and render unique content.

‚úÖ Two steps: pass then receive

To use props, you do it in two steps:

  1. Pass props into the component when you render it (in the parent).
  2. Receive props inside the child component using the props parameter.

Rule ‚úÖ

If you pass props but don‚Äôt receive them, the child won‚Äôt know the values exist.

üì§ Step 1: passing props (like attributes)

In JSX, props look like HTML attributes. The lecture passes name, ingredients, photoName, and price. 

function Menu() {
  return (
    <div>
      <Pizza
        name="Pizza Spinaci"
        ingredients="Tomato, mozarella, spinach, and ricotta cheese"

      />
    </div>
  );

Ordering note üßæ

The order of props in JSX does not matter. You can write price before photoName, etc.

üì• Step 2: receiving props (props parameter)

The child component receives props by accepting a parameter (commonly named props). 

function Pizza(props) {
  console.log(props);

  return (
    <div>
      <h3>Pizza</h3>
    </div>
  );
}

What is props exactly? üß†

Props is a single object built by React from the attributes you passed into <Pizza />. Example keys: props.name, props.photoName, props.ingredients, props.price.

üß© Using props in JSX

To use values from the props object, you must enter JavaScript mode in JSX using { }. 

function Pizza(props) {
  return (
    <div>
      <img src={props.photoName} alt={props.name} />
      <h3>{props.name}</h3>
      <p>{props.ingredients}</p>
      <span>{props.price}</span>
    </div>
  );
}

Reminder ‚úÖ

In JSX: strings can be written as "text", but values like variables/expressions must be inside { }.

üî¢ Important: strings vs numbers in props

If you pass a number as a string (like price="10"), JavaScript treats it as text. Then math becomes string concatenation.

Wrong for math ‚ùå

// price is a STRING
<Pizza price="10" />

// later
{props.price + 3} // becomes "103"

Because "10" + 3 ‚Üí "103" (string concatenation).

Correct ‚úÖ

// price is a NUMBER
<Pizza price={10} />

// later
{props.price + 3} // becomes 13

Because 10 + 3 ‚Üí 13 (numeric addition).

Rule of thumb ‚úÖ

If the prop is not a plain string, wrap it in { }.

üß∞ You can pass anything as a prop

Props are extremely flexible. You can pass:

  • üßæ Strings
  • üî¢ Numbers
  • üìö Arrays
  • üß± Objects
  • üß† Functions (later: event handlers)
  • üß© Even other components / JSX

Mini examples ‚úÖ

Important ‚úÖ

Props flow in one direction: parent ‚Üí child. (This is called ‚Äúone-way data flow‚Äù.)

Quick summary üîë

Props are how React components receive input data from their parent. You pass props like attributes, React bundles them into a props object, then you read them using props.name, etc. Wrap non-strings (numbers, arrays, objects, functions) in { } when passing.

ğŸ”— Relationships (quick beginner primer)

When multiple tables exist, you usually connect them using IDs. This avoids repeating the same data everywhere.

Example: One booking belongs to one cabin

A bookings table can store cabinId instead of copying cabin details into every booking row.

Example: One guest can have many bookings

A bookings row can store guestId. Many bookings can point to the same guest.

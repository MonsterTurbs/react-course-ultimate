3) Generate 1‚Äì20 Options Dynamically (Array.from Trick) üó∫Ô∏è

Problem: You don‚Äôt want to write 20 <option> tags by hand. üòÖ

Solution: generate an array [1,2,...,20] and map it into options.

A) Create numbers 1..20 using Array.from

// Creates: [1, 2, 3, ... , 20]
const numbers = Array.from({ length: 20 }, (_, i) => i + 1);

You don‚Äôt need to memorize how Array.from works internally; just know this pattern is common in React.

B) Map numbers to <option> elements

<select>
  {numbers.map((num) => (
    <option value={num} key={num}>
      {num}
    </option>
  ))}
</select>

Why do we add key? üîë

Whenever you render a list in React (options, items, rows, etc.), React expects a stable unique key. Here, num is unique, so it‚Äôs a good key.

5) Handle Form Submission (onSubmit) üñ±Ô∏è

A) Create a handler function

  // logic later
}

B) Listen for the submit event on the form

  ...
</form>

Important: you pass the function reference (handleSubmit), you do not call it (handleSubmit()). React will call it when the event occurs.

When does submit happen?

  • Click the submit button ‚úÖ
  • Press Enter while focused in the input ‚úÖ

This is normal HTML form behavior‚Äîand it‚Äôs why onSubmit is better than a button onClick.

If you use button onClick only: it won‚Äôt catch Enter submissions. üò¨

6) Prevent Page Reload (Single-Page App Behavior) ‚õî

Default HTML behavior: submitting a form reloads the page.

In React SPAs, you usually want to submit without reloading.

Fix: call preventDefault()

function handleSubmit(e) {
  e.preventDefault();
  // now the page will not reload
}

Why does e exist? üß™

When the submit event happens, React calls your handler and passes the event object into it. This is similar to Vanilla JavaScript.

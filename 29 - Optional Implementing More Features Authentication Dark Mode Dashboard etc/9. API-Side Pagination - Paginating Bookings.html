1) Pagination needs the total number of results

The Pagination component needs a count so it can calculate pages and render ‚ÄúX results‚Äù. You could compute bookings.length, but that only works for the current page. A better approach is to ask Supabase for the exact total count.

‚úÖ Supabase feature: select(..., { count: "exact" })

You can pass a second argument to select to request a count without relying on array length. Useful when you need total results for pagination. 

// services/apiBookings.js (conceptual)
export async function getBookings({ filter, sortBy, page }) {
  let query = supabase
    .from("bookings")
    .select("*, cabins(name), guests(fullName)", { count: "exact" });

  // ...filter + sort...

  const { data, error, count } = await query;

  if (error) throw error;

  return { data, count };
}

Update the consumer code to match new return shape

Since the service now returns { data, count }, you destructure and rename: 

// hooks/useBookings.js (conceptual)
const { data: { data: bookings, count } = {} } = useQuery(...);

‚ö†Ô∏è Common pitfall

On first render, query data is undefined. Destructuring will crash unless you provide a default object: data: { ... } = {}.

2) Get the current page from the URL (search params)

Pagination state lives in the URL (e.g., ?page=3), so the hook needs to read it. You already have logic for reading filters and sorting; now you add page as well. 

// hooks/useBookings.js (conceptual)
import { useSearchParams } from "react-router-dom";

const [searchParams] = useSearchParams();
const page = Number(searchParams.get("page")) || 1;

‚úÖ Important

Add page to your React Query key. Changing page should refetch data automatically. 

// queryKey example
queryKey: ["bookings", filter, sortBy, page]

3) Server-side pagination with Supabase range(from, to)

React Query refetching on page change is only half the work. You also need to fetch the correct slice from the database. Supabase provides range(from, to).

üéØ Example (page size = 10)

  • Page 1 ‚Üí results 0 to 9
  • Page 2 ‚Üí results 10 to 19
  • Page 3 ‚Üí results 20 to 29

Compute from/to

// utils/constants.js
export const PAGE_SIZE = 10;
// services/apiBookings.js (conceptual)
import { PAGE_SIZE } from "../utils/constants";

if (page) {
  const from = (page - 1) * PAGE_SIZE;
  const to = from + PAGE_SIZE - 1;

  query = query.range(from, to);
}

üß† Why a constants file?

  • Prevents ‚Äúmagic numbers‚Äù duplicated in multiple files.
  • Changing page size becomes a one-line change.
  • Avoids maintenance nightmares.

4) Verify the behavior (expected results)

  • Page 1 returns the first PAGE_SIZE bookings.
  • Page 2 returns the next PAGE_SIZE bookings.
  • Last page may return fewer results (e.g., 4 results remaining).
  • Count remains the full total (e.g., 24 bookings).

How to navigate between pages in a Single Page Application (SPA) without hard reloads.

Topic: React Router v6 Goal: SPA navigation Tools: <Link>, <NavLink>

üéØ What you‚Äôre building in this lecture

  • Replace normal <a href> links (hard reload) with React Router navigation (no reload).
  • Create a reusable navigation component (PageNav) you can place on multiple pages.

‚úÖ Key idea

In an SPA, navigation should update the URL without reloading the page. React Router does this by intercepting clicks and swapping components in the DOM.

üß† Why <a href> is not ideal for SPAs

If you add a normal anchor tag like:

<a href="/pricing">Go to Pricing</a>

‚ö†Ô∏è What happens

  • The browser performs a full page reload.
  • You‚Äôll see new requests in the Network tab (HTML/JS/CSS/assets may re-fetch).
  • The SPA ‚Äúnative-app feel‚Äù is lost because navigation becomes slower and resets runtime state.

Tip: This is why you saw multiple network requests after clicking an anchor link.

‚úÖ The solution: React Router‚Äôs Link

React Router provides <Link> which changes the URL and renders a new route without reloading the page.

Basic example (Home ‚Üí Pricing)

// HomePage.jsx
import { Link } from "react-router-dom";

export default function HomePage() {
  return (
    <div>
      <h1>WorldWise</h1>
      <Link to="/pricing">Go to Pricing</Link>
    </div>
  );
}

üß© Important detail

Use a leading slash: to="/pricing" so the path is always absolute (from the root), not relative.

‚úÖ Result

Clicking the link updates the URL and swaps the rendered component. The Network tab stays quiet (no ‚Äúhard refresh‚Äù).

üß± Reusable navigation: PageNav component

Instead of adding links manually on every page, create a reusable navigation component and render it where you need it.

Example structure

  • src/pages/ ‚Üí route pages (HomePage, Pricing, Product, etc.)
  • src/components/ ‚Üí reusable UI pieces (PageNav, buttons, layout)

PageNav example

// components/PageNav.jsx
import { Link } from "react-router-dom";

export default function PageNav() {
  return (
    <nav>
      <ul>
        <li><Link to="/">Home</Link></li>
        <li><Link to="/pricing">Pricing</Link></li>
        <li><Link to="/product">Product</Link></li>
      </ul>
    </nav>
  );
}

Use it in each page

// pages/Pricing.jsx
import PageNav from "../components/PageNav";

export default function Pricing() {
  return (
    <>
      <PageNav />
      <h1>Pricing</h1>
    </>
  );
}

üß† Concept

You can render navigation inside each page, or once at a higher level (like in App) depending on your layout strategy.

‚≠ê Highlight the active page using NavLink

In menus, it‚Äôs common to highlight the current page. React Router makes this easy with <NavLink>.

Replace Link ‚Üí NavLink

// components/PageNav.jsx
import { NavLink } from "react-router-dom";

export default function PageNav() {
  return (
    <nav>
      <ul>
        <li><NavLink to="/">Home</NavLink></li>
        <li><NavLink to="/pricing">Pricing</NavLink></li>
        <li><NavLink to="/product">Product</NavLink></li>
      </ul>
    </nav>
  );
}

‚úÖ What you get automatically

The currently active link receives a CSS class (commonly active). You can style it later to highlight the current route.

Later, once you add CSS (including CSS Modules), you can style that active state (color, underline, background, etc.).

üåê Why URLs matter in SPAs

  • üîÅ Refresh-safe: If you refresh on /pricing, you still land on Pricing.
  • üîñ Bookmarkable: Users can bookmark a specific screen.
  • ü§ù Shareable: You can send a link to a teammate and they open the exact same view.

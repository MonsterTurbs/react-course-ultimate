1) The Classic Question

2) Definitions (Short and Precise)

üß†

State

Internal data owned by the component where it is declared. It can change over time.

üì¶

Props

External data owned by a parent component and passed down to a child component.

Quick comparison table (mentally)

  • Ownership: State = owned by the component. Props = owned by the parent.
  • Mutability: State = can be updated by the owning component. Props = read-only for the receiving component.
  • Purpose: State = interactivity. Props = configuration + communication parent ‚Üí child.

3) Helpful Mental Models

Props = function parameters

Think of a component like a function. Props are like the inputs/arguments that configure how it behaves.

State = component memory

State persists across re-renders. It is the ‚Äúmemory‚Äù of the component that stores values over time.

Rule of thumb: If the UI needs to change because of user actions, you likely need state. ‚úÖ

4) Re-Render Behavior (Where State and Props Connect) üîÅ

State updates trigger re-renders

When a component updates its own state (via its setter), React re-renders that component. This is how interactivity happens.

New props also trigger re-renders

Even though props are read-only inside the child, if the child receives updated props, the child will re-render too.

Key connection: If state is passed down as props, then updating that state re-renders both components. ‚úÖ

  • The component that owns the state re-renders (because state changed).
  • The child component also re-renders (because it received new props).

Example scenario from the lecture

A parent component has upVotes state (created with useState) and passes it into a child as a prop. When upVotes changes, the parent re-renders, and the child re-renders to stay in sync. 

// Parent
const [upVotes, setUpVotes] = useState(0);

return <Question upVotes={upVotes} />;

// Child
function Question({ upVotes }) {
  return <Button label={upVotes} />;
}

If the child didn‚Äôt re-render, it would show the old number and the UI would be out of sync.

5) What Each One Is Used For

üïπÔ∏è

State: make components interactive

State is used when the UI needs to change over time (clicks, typing, toggles, fetching data, etc.).

‚öôÔ∏è

Props: configure child components

Props act like ‚Äúsettings‚Äù that a parent uses to control how a child looks/behaves.

Simple phrasing: State drives behavior; props drive configuration. ‚úÖ

6) Interview-Ready Answer (1‚Äì2 Sentences) üß™

State is internal data owned by a component that can change over time and triggers re-renders. Props are read-only inputs passed from parent to child to configure the child; when props change, the child re-renders.

Extra credit: When state is passed as props, updating the state re-renders the parent and the child. üôÇ

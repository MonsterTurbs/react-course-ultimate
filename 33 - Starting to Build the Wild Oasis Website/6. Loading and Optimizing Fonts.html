Quick mental model

Next.js gives you a font loader function (e.g., Josefin_Sans). You call it with options, it returns an object with a generated className. You add that className to your HTML (commonly the <body>) so the font applies.

üõ†Ô∏è Step-by-step: Add ‚ÄúJosefin Sans‚Äù with next/font/google

1) Import the font loader

import { Josefin_Sans } from "next/font/google";

2) Configure the font

Most common options:

  • subsets: Which character set you need (e.g., "latin")
  • display: Font display strategy (commonly "swap")
  • weight: Required for non-variable fonts (variable fonts often don‚Äôt need it)
const josefin = Josefin_Sans({
  subsets: ["latin"],
  display: "swap",
  // weight: ["400", "700"], // only if the font is NOT variable
});

3) Apply the generated className

The josefin object contains a Next.js-generated class name (e.g., josefin.className). Apply it to <body> to use the font everywhere. 

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body className={`${josefin.className}`} >
        {children}
      </body>
    </html>
  );
}

‚úÖ Result: Your pages render using the Josefin Sans font, and Next.js self-hosts the font files for you.

üåê In a Client Component

üè∑Ô∏è Using multiple fonts (common real-world pattern)

Sometimes you want one font for body text and another for headings. You can import two fonts and apply their classNames to different elements. 

import { Josefin_Sans, Roboto_Slab } from "next/font/google";

const bodyFont = Josefin_Sans({ subsets: ["latin"], display: "swap" });
const headingFont = Roboto_Slab({ subsets: ["latin"], display: "swap" });

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body className={bodyFont.className}>
        <h1 className={headingFont.className}>Heading font</h1>
        {children}
      </body>
    </html>
  );
}

üß† Remember: you don‚Äôt have to apply the font to <body>. Apply it only where needed.

üìå Local fonts (brief mention)

Next.js can also load fonts that you ship with your project using next/font/local. This is useful if you have a licensed font file.

  src: "./path-to-font/MyFont.woff2",
  display: "swap",
});

‚úÖ Same idea: configure ‚Üí get className ‚Üí apply to elements.

üßØ Common mistakes & quick fixes

  • ‚ùå Font imported but not applied: You forgot className={font.className}.
  • ‚ùå Wrong import path: Use next/font/google (Google fonts) or next/font/local (local fonts).
  • ‚ùå Missing subset: Set subsets: ["latin"] (or the subset you need).
  • ‚ùå Non-variable font weight not set: Add weight (e.g., ["400","700"]) if required.
  • ü§î Where is my console.log? If you log in a server component, check the terminal, not the browser.

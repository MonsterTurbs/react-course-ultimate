üö® Common pitfall: w-full breaks flex-item behavior

In the lecture, one row didn‚Äôt align correctly. The root cause was an input utility class that contained w-full.

  • If an element is a flex item, w-full can cause it to behave unexpectedly and fight Flexbox sizing.
  • Fix: remove w-full from the shared .input class, then add w-full only where the input is not a flex item (e.g., inside a wrapper div).
// index.css (Tailwind layer) ‚Äî BEFORE (problematic)
.input {
  @apply ... w-full ...;
}

// AFTER (better for flex layouts)
.input {
  @apply ...; /* remove w-full */
}

// In components where input is NOT a flex item
<input className="input w-full" />
‚úÖ Practical rule:
If your input is directly inside a Flex row, try grow instead of w-full.

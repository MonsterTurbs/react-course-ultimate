üß† Props object recap (it always exists)

Whenever a component is rendered, React calls it like a normal function and provides one argument: a props object. That object contains all props passed from the parent.

Important note üìù

Even if you don‚Äôt pass any props, the props object still exists‚Äîit's just empty.

‚úÖ Solution: destructure in the function parameter

Instead of receiving props, you can destructure the specific props you need directly in the parameter list. The destructured names must match the prop names you pass from the parent.

Before ‚Üí After üß©

// BEFORE
function Pizza(props) {
  console.log(props);
  return <div>{props.pizzaObj.name}</div>;
}

// AFTER (destructured)
function Pizza({ pizzaObj }) {
  console.log(pizzaObj);
  return <div>{pizzaObj.name}</div>;
}

Cleaner JSX ‚úÖ

Now you can use pizzaObj.name instead of props.pizzaObj.name.

üö´ Common beginner mistakes + checklist

Top mistake: forgetting the curly braces ‚ö†Ô∏è

Without { }, you are NOT destructuring.

// ‚ùå Not destructuring

// ‚úÖ Destructuring
function Order({ closeHour }) { ... }

Quick checklist ‚úÖ

  • Do destructured names match the prop names passed from the parent?
  • Did you include curly braces: function Comp({ ... })?
  • If a prop might be missing, do you handle undefined gracefully?

Course standard ‚úÖ

From this point onward, you will usually see props received like: function Comp({ propA, propB }).

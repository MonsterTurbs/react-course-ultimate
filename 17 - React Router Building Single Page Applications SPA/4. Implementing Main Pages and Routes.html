üéØ What you‚Äôre building

You will set up main page routes so different URL paths render different React page components: / (Home), /product (Product), /pricing (Pricing), and a catch-all * (Not Found).

‚úÖ Key idea

With routing, you match a URL path to a React component. React Router watches the current URL and renders the matching route‚Äôs element.

üß† Core concepts (simple explanations)

Concept Meaning (Beginner-friendly)
BrowserRouter The ‚Äúrouter wrapper‚Äù that enables client-side routing using the browser‚Äôs history + URL bar. It should usually wrap your routes (often at the top level).
Routes A container for multiple route definitions. React Router uses it to choose which single Route matches.
Route One mapping: path ‚Üí element. Example: path="/product" renders <Product />.
element prop You pass a real React element (like <Product />), not just the component name. This makes it easy to pass props later if needed.
path="*" Catch-all route. If nothing else matches, show a ‚ÄúPage Not Found‚Äù screen.

üß± Step-by-step setup

1) Create a pages/ folder and page components

‚ÄúPages‚Äù are usually top-level screens matched to URLs (Home, Product, Pricing, etc.). Keep them in a dedicated folder. 

// src/pages/Homepage.jsx

  return <h1>WorldWise</h1>;

// src/pages/Product.jsx

  return <h1>Product</h1>;
}

// src/pages/Pricing.jsx

  return <h1>Pricing</h1>;
}

// src/pages/PageNotFound.jsx
export default function PageNotFound() {
  return <h1>Not found</h1>;
}

2) Install React Router DOM (v6)

üì¶ Install

Use react-router-dom (not just react-router) because you‚Äôre routing in the browser (DOM).

npm install react-router-dom@6

3) Define routes in your App component

Wrap your routes with BrowserRouter, then define each route with Route inside Routes. 

import { BrowserRouter, Routes, Route } from "react-router-dom";

import Homepage from "./pages/Homepage";
import Product from "./pages/Product";
import Pricing from "./pages/Pricing";
import PageNotFound from "./pages/PageNotFound";

export default function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Homepage />} />
        <Route path="/product" element={<Product />} />
        <Route path="/pricing" element={<Pricing />} />
        <Route path="*" element={<PageNotFound />} />
      </Routes>
    </BrowserRouter>
  );
}
üß© What happens at runtime?
  1. React Router checks the current URL (example: /product).
  2. It scans your route list and finds the best match (path /product).
  3. It renders that route‚Äôs element (here: <Product />).
  4. All other routes are ignored for that render.

Example B: ‚ÄúNot found page‚Äù

If the user types a URL you didn‚Äôt define (e.g., /abc), the catch-all * route displays.

‚úÖ Best practice

Always add a * route so users don‚Äôt see a blank screen or confusing errors.

‚ö†Ô∏è Common mistakes (and fixes)

  • Installing the wrong package: using react-router only.
    Fix: install react-router-dom for browser apps.
  • Forgetting imports: missing Route or Routes in the import line.
    Fix: import { BrowserRouter, Routes, Route } from "react-router-dom";
  • Using component prop (old versions): in v6 you use element.
    Fix: <Route element={<MyPage />} />
  • Missing the leading slash: writing path="product" instead of "/product".
    Fix: main routes usually start with /.

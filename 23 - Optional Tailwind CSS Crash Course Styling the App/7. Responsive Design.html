üìå The core concept: Mobile-first (min-width)

Tailwind‚Äôs responsive system is mobile-first. That means: unprefixed classes are your default styles (good for small screens), and breakpoint-prefixed classes override those defaults once the viewport is wide enough.

üß© How breakpoint prefixes work

You can prefix almost any Tailwind utility class with a breakpoint: sm:, md:, etc.

Example 1: Override spacing for larger screens

Start with a mobile-friendly spacing, then increase it for wider viewports. 

<div className="my-10 sm:my-16">
  ...
</div>
  • my-10 applies everywhere by default.
  • sm:my-16 overrides it only when viewport is ‚â• 640px.

Example 2: Responsive padding on a header

  ...
</header>

On mobile: horizontal padding is px-4. On small screens and up: it becomes px-6.

üß∞ Custom breakpoints (real projects)

    },
  },

Then you can write: tablet:px-6, laptop:grid, etc.

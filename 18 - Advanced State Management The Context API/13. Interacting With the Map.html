üéØ Goal and big idea

We already store lat and lng in the URL as a Query string (e.g. ?lat=38.72&lng=-9.13). Now we will:

  • ‚úÖ Re-center the Leaflet map when the URL lat/lng changes.
  • ‚úÖ Remember the last map position even if the URL loses lat/lng when navigating back.
  • ‚úÖ Handle clicks on the map: navigate to the form route.
  • ‚úÖ Include the clicked coordinates in the URL so the form can read them later.

üß† Key concept: Leaflet map behaviors are often implemented via custom components + hooks (e.g. useMap, useMapEvents), not by random props.

Part A ‚Äî Move the map using Query String

1) Read lat and lng from the URL

In the Map component, we read query parameters using React Router‚Äôs useSearchParams(). This gives us a searchParams object where we can call .get("lat") and .get("lng"). 

import { useSearchParams } from "react-router-dom";

const [searchParams] = useSearchParams();
const mapLat = searchParams.get("lat");
const mapLng = searchParams.get("lng");

‚ö†Ô∏è Important: searchParams.get() returns a string or null. Many times you‚Äôll want Number(mapLat) / Number(mapLng).

2) Why center doesn‚Äôt ‚Äúreact‚Äù by itself

In React-Leaflet, the <MapContainer center={...} /> prop is used for the initial setup. When your center value changes later, Leaflet does not automatically pan to it. So we implement it ourselves.

3) Create a custom component to re-center the map

We create a component (example name: ChangeCenter) and use the Leaflet hook useMap(). That hook returns the map instance, so we can call map.setView(position). 

import { useMap } from "react-leaflet";

function ChangeCenter({ position }) {
  const map = useMap();
  map.setView(position); // optional: map.setView(position, 6)
  return null;
}

4) Remember the last position (so the map doesn‚Äôt ‚Äújump back‚Äù)

When you navigate back to the list, the URL may no longer have lat and lng. If you immediately fall back to a default center (like [40, 0]), the map will jump. Instead, store the map position in local state and synchronize it when URL params exist. 

import { useEffect, useState } from "react";

const [mapPosition, setMapPosition] = useState([40, 0]);

useEffect(function () {
  if (mapLat && mapLng)
    setMapPosition([Number(mapLat), Number(mapLng)]);
}, [mapLat, mapLng]);

‚úÖ Why this works: when you return to the list and mapLat/mapLng become null, the effect does nothing, so the last mapPosition remains.

5) Use your custom component inside the map

return (
  <div className={styles.mapContainer}>
    <MapContainer center={mapPosition} zoom={6} className={styles.map}>
      <ChangeCenter position={mapPosition} />
      // ...TileLayer, markers, popups...
    </MapContainer>
  </div>
);

üß† Reminder: useEffect is often a synchronization mechanism‚Äîhere we synchronize URL state ‚Üî map state.

Part B ‚Äî Detect map clicks and navigate programmatically

1) Why we can‚Äôt just do onClick on MapContainer

React-Leaflet uses its own event system. For map events, we typically create a small component and attach listeners using useMapEvents().

2) Create a DetectClick component

We use React Router‚Äôs useNavigate() to move to the form route ("form") without clicking a link. 

import { useNavigate } from "react-router-dom";
import { useMapEvents } from "react-leaflet";

function DetectClick() {
  const navigate = useNavigate();

  useMapEvents({
    click: (e) => {
      navigate("form");
    },
  });

  return null;
}

3) Mount the component inside MapContainer

<MapContainer center={mapPosition} zoom={6} className={styles.map}>
  <ChangeCenter position={mapPosition} />
  <DetectClick />
  // ...layers/markers...
</MapContainer>

Part C ‚Äî Put clicked position in the URL (lat/lng)

When the user clicks on the map, we want the form to know where that click happened. Instead of inventing temporary global state, we store it in the URL as a Query string.

1) Read the coordinates from the Leaflet click event

Leaflet‚Äôs click event contains e.latlng (not the usual DOM event). It has lat and lng.

2) Navigate to the form with a Query string

function DetectClick() {
  const navigate = useNavigate();

  useMapEvents({
    click: (e) => {
      const lat = e.latlng.lat;
      const lng = e.latlng.lng;

      navigate(`form?lat=${lat}&lng=${lng}`);
    },
  });

  return null;
}

üîó Why this is powerful: the form can later read lat and lng using useSearchParams(), just like the map did. The URL becomes a lightweight ‚Äúglobal state‚Äù channel between pages.

3) Expected user experience

  • üñ±Ô∏è Click a city in the list ‚Üí URL updates with city ID and lat/lng ‚Üí map re-centers.
  • ‚¨ÖÔ∏è Navigate back to list ‚Üí URL loses lat/lng, but map stays where it was (because state remembered it).
  • üó∫Ô∏è Click anywhere on the map ‚Üí navigate to form and include clicked lat/lng in the URL.

‚ö†Ô∏è Common pitfalls (and quick fixes)

  • Hook order rule: don‚Äôt return early before hooks (e.g., don‚Äôt render conditionally above useSearchParams()).
  • Null query params: searchParams.get("lat") can be null when the URL doesn‚Äôt have it. Guard with if (mapLat && mapLng).
  • Map ‚Äújumping‚Äù: if you always fall back to a default center, the map will snap back when leaving the city route. Keep a local mapPosition state and only update it when params exist.
  • Types: query params are strings. Convert to numbers before using as coordinates.
  • React-Leaflet pattern: advanced behavior often requires ‚Äúhelper components‚Äù that return null but attach logic with hooks.

  • üñ±Ô∏è DetectClick + useMapEvents() handles map clicks.

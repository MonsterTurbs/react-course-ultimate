Lecture reviewer: build interactive flashcards using a single piece of state (selectedId) and practice list rendering + events.

1) What You‚Äôre Building

Goal: A flashcard grid where each card shows a question by default. When you click a card, it flips to show the answer. üôÇ

  • Click a card ‚Üí it opens (shows answer)
  • Click another card ‚Üí the previous one closes, the new one opens
  • Click the same open card again ‚Üí it closes (back to question)
üß©

Key idea

You can control a whole UI with one piece of state that stores the ‚Äúcurrently selected card ID‚Äù.

2) Starter Data (Questions Array)

You get an array of objects. Each object has:

  • id (unique)
  • question
  • answer

Only the question is visible by default.

3) Step 1 ‚Äî Render the Cards (Static First) üó∫Ô∏è

Map over the questions array

You already know the pattern: JavaScript mode ‚Üí map ‚Üí return JSX.

function Flashcards() {
  return (
    <div className="flashcards">
      {questions.map((question) => (
        <div key={question.id}>
          <p>{question.question}</p>
        </div>
      ))}
    </div>
  );
}

Don‚Äôt forget: add className="flashcards" to the container so your CSS grid styling works. üß±

Also copy style.css if you‚Äôre running it outside CodeSandbox.

Fix the key warning üîë

Every element rendered inside a list needs a unique key. Here, the best key is question.id.

4) Step 2 ‚Äî Add One Piece of State: selectedId üß†

State meaning: which card is currently open?

import { useState } from "react";

function Flashcards() {
  const [selectedId, setSelectedId] = useState(null);
  // null means: no card is open

  return (...);
}

Why null? ‚úÖ

It represents ‚Äúnothing selected‚Äù (all cards show questions).

5) Step 3 ‚Äî Use State to Decide What to Show (Question vs Answer) üéØ

While mapping, compare question.id to selectedId. If they match, show the answer; otherwise show the question. 

{questions.map((question) => (
  <div key={question.id}>
    <p>
      {question.id === selectedId
        ? question.answer
        : question.question}
    </p>
  </div>
))}

Conditional styling (selected class) üé®

Also apply the CSS class selected when the card is open:

<div
  key={question.id}
  className={question.id === selectedId ? "selected" : ""}
>
  ...
</div>

Why this works: each mapped card has access to its own question object, so each card can compare its ID to the selected ID. ‚úÖ

6) Step 4 ‚Äî Update State on Click üñ±Ô∏è

A) Create a click handler that receives an ID

function handleClick(id) {
  setSelectedId(id);
}

But we need to pass the clicked card‚Äôs ID into the handler.

Important React rule: event handlers must receive a function, not a function call. üö´

Wrong: onClick={handleClick(question.id)} (runs immediately)
Correct: onClick={() => handleClick(question.id)}

B) Attach it to the card

<div
  key={question.id}
  onClick={() => handleClick(question.id)}
>
  ...
</div>

Result: clicking different cards switches the open card. ‚úÖ

But clicking the same card again still doesn‚Äôt close it yet ‚Äî that‚Äôs the next step.

7) Step 5 ‚Äî Toggle the Same Card (Open ‚Üî Close) üîÅ

If you click an already-open card, set selectedId back to null. Otherwise set it to the clicked card ID. 

function handleClick(id) {
  setSelectedId(id !== selectedId ? id : null);
}

Behavior achieved:

  • Click closed card ‚Üí opens it (selectedId becomes that id)
  • Click a different card ‚Üí switches selection
  • Click the same open card ‚Üí closes it (selectedId becomes null)

8) Full Minimal Version (Putting It All Together) ‚úÖ

import { useState } from "react";

export default function Flashcards({ questions }) {
  const [selectedId, setSelectedId] = useState(null);

  function handleClick(id) {
    setSelectedId(id !== selectedId ? id : null);
  }

  return (
    <div className="flashcards">
      {questions.map((question) => (
        <div
          key={question.id}
          className={question.id === selectedId ? "selected" : ""}
          onClick={() => handleClick(question.id)}
        >
          <p>
            {question.id === selectedId
              ? question.answer
              : question.question}
          </p>
        </div>
      ))}
    </div>
  );
}

Study note: This exercise is a great example of ‚Äústate modeling‚Äù. Instead of many booleans, you store a single selected ID. üéØ

9) Quick Self-Check üß†

  1. Why is selectedId enough to control all cards?
  2. Why do we use null as the initial state?
  3. What is wrong with onClick={handleClick(question.id)}?
  4. Explain the toggle logic: id !== selectedId ? id : null.
  5. What is the role of the key prop in the map?

Answer hints ‚úÖ

  • Only one card can be open, so one ID models the entire UI state.
  • null means ‚Äúno selection‚Äù.
  • It runs immediately instead of waiting for the click event.
  • If clicking a different card, open it; if clicking the same card, close it.
  • Keys help React track list items between renders.

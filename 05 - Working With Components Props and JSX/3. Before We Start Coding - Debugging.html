When you make certain mistakes, Create React App displays a full-screen overlay with the error details. This overlay is often the fastest way to understand what went wrong.

üß≠ ESLint: your early warning system

üß† React 18 vs React 17 root rendering (common confusion)

If you ever work with older codebases, you may see different root rendering code. In React 18, createRoot is the standard approach.

React 18 style 
import React from "react";
import ReactDOM from "react-dom/client";

import App from "./App";

const root = ReactDOM.createRoot(document.getElementById("root"));
root.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
React 17 style (older) 
import React from "react";
import ReactDOM from "react-dom";

import App from "./App";

ReactDOM.render(<App />, document.getElementById("root"));

Important: the older API needs the target DOM node as the second argument.

How to check your React version

Look in package.json under dependencies (for example, you might see React 18.x).

üß∑ Strict Mode (quick note)

When enabled during development, <React.StrictMode> can cause components to render twice. This is intentional: it helps React detect certain side-effect bugs and outdated API usage.

Remember

Double renders in development are not necessarily a bug. Always confirm behavior in production builds if unsure.

üéØ Lesson objective

üß† Why this needs a prepare function

In this slice, createCustomer needs two inputs: fullName and nationalId. Redux Toolkit action creators normally accept one payload.

‚úÖ The prepare method lets you:
  • Accept multiple arguments
  • Return a single standardized payload object

Also: where to put ‚Äúside effects‚Äù (like dates / random IDs)

Creating timestamps (like createdAt) is a small side effect. You should not generate this inside the reducer. Put it in prepare instead.

‚ö†Ô∏è Reducers should be ‚Äúpure‚Äù (no random values, no new Date, no API calls). If you need computed values, do them in prepare or in thunks.

üß© Core structure of a customer slice (Redux Toolkit)

This is the exact pattern the lecture is teaching:

// customerSlice.js (template)
import { createSlice } from '@reduxjs/toolkit';

const initialState = {
  fullName: '',
  nationalId: '',
  createdAt: '',
};

const customerSlice = createSlice({
  name: 'customer',
  initialState,
  reducers: {
    createCustomer: {
      // 1) Reducer: receives state + action
      reducer(state, action) {
        state.fullName = action.payload.fullName;
        state.nationalId = action.payload.nationalId;
        state.createdAt = action.payload.createdAt;
      },

      // 2) Prepare: builds the action payload
      prepare(fullName, nationalId) {
        return {
          payload: {
            fullName,
            nationalId,
            createdAt: new Date().toISOString(),
          },
        };
      },
    },

    updateName(state, action) {
      state.fullName = action.payload;
    },
  },
});

export const { createCustomer, updateName } = customerSlice.actions;
export default customerSlice.reducer;
‚úÖ Key takeaway: the reducer receives (state, action). The payload is inside action.payload, which is produced by prepare().

üìù Method syntax note (modern JavaScript)

The lecture mentions that writing methods like this: 

prepare(fullName, nationalId) {
  // ...
}

is equivalent to: 

prepare: function (fullName, nationalId) {
  // ...
}

Same functionality, just a modern shorthand.

üì§ Export checklist (must be correct)

  1. Export action creators from customerSlice.actions
  2. Export the reducer as default from customerSlice.reducer
‚ö†Ô∏è Common bug: using customerSlice.action (singular) instead of customerSlice.actions (plural). That will make it ‚Äúundefined‚Äù.

  • üè∑Ô∏è Reducers read data via action.payload.

1) What are API Routes?

In the Pages Router, an API Route is a file-based backend endpoint (server code) that lives under pages/api.

‚≠ê Why they matter

  • They run on the server (not in the browser).
  • They are the most common way to do data mutations (create/update/delete) in the Pages Router ‚úÖ
  • They are similar to Route Handlers in the App Router, but used more heavily here because the Pages Router has no Server Actions.

üß† Mental model

Think of an API Route as a tiny ‚Äúbackend function‚Äù inside your Next.js project‚Äîlike a mini Express route. Your React components will call it using fetch().

2) Creating an API Route

API Routes must be placed in the special folder: pages/api.

Each file in that folder becomes an endpoint:

  • pages/api/contact.js ‚Üí /api/contact
  • pages/api/users.js ‚Üí /api/users

Basic skeleton

// pages/api/contact.js
export default async function handler(req, res) {
  // server-side code here
  return res.status(200).json({ ok: true, message: "Hello from API Route" });
}

‚úÖ Key difference vs App Router Route Handlers

  • In Pages Router: you export one default handler.
  • In App Router: you export named functions like GET, POST, etc.

3) Request and Response (req, res)

The handler receives:

üì• req ‚Äî incoming request

  • req.method (GET, POST, etc.)
  • req.body (data sent by the client, usually in POST)
  • req.query (URL query params)

üì§ res ‚Äî response you send back

  • res.status(200) sets HTTP status code
  • res.json({...}) sends JSON

üîé Common HTTP status codes

  • 200 OK ‚Äî request succeeded ‚úÖ
  • 405 Method Not Allowed ‚Äî wrong HTTP method used üö´
  • 500 Internal Server Error ‚Äî unexpected server failure üí•

5) Handling POST-only requests (Guard Clause)

You usually want this endpoint to accept POST only (because POST can include a request body). 

export default async function handler(req, res) {
  if (req.method !== "POST") {
    return res.status(405).json({
      success: false,
      message: "Please make a POST request."
    });
  }

  // ...handle POST logic below
}

‚ö†Ô∏è Important detail

Use return when sending the response early‚Äîotherwise the function might continue and still run the mutation.

6) Inserting data into Supabase + Error Handling

On success, return a friendly JSON response. On failure, return an error response.

Example API Route (contact submission)

// pages/api/contact.js
// Assumes you already created and exported a Supabase client (e.g., from lib/supabase)

import supabase from "../../lib/supabase";

export default async function handler(req, res) {
  // 1) Only allow POST
  if (req.method !== "POST") {
    return res.status(405).json({
      success: false,
      message: "Please make a POST request."
    });
  }

  // 2) Read data from the request body
  // (Client must send JSON body. Example shown later.)
  const { fullName, email, subject, message } = req.body || {};

  // 3) Minimal validation (optional but recommended)
  if (!fullName || !email || !subject || !message) {
    return res.status(400).json({
      success: false,
      message: "Missing required fields."
    });
  }

  // 4) Insert into DB
  const { error } = await supabase
    .from("contact")
    .insert([
      {
        full_name: String(fullName).slice(0, 200),
        email: String(email).slice(0, 200),
        subject: String(subject).slice(0, 200),
        message: String(message).slice(0, 2000)
      }
    ]);

  // 5) Respond
  if (error) {
    return res.status(500).json({
      success: false,
      message: "Could not send your message. Please try again."
    });
  }

  return res.status(200).json({
    success: true,
    message: "Thanks for your message. We will be in touch soon."
  });
}

üß© Why insert([ ... ]) uses an array

Supabase‚Äôs insert expects an array so you can insert multiple rows at once. Even if you insert a single row, you still wrap it in an array.

üîê Security note (important)

Client-side validation is not enough. A malicious user can bypass UI restrictions. Always validate and authorize on the server for real production apps.

7) How the client calls the API Route

In the Pages Router, your React components must call API Routes manually (unlike Server Actions).

Example: calling /api/contact with fetch

async function submitContact(formValues) {
  const res = await fetch("/api/contact", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(formValues)
  });

  const data = await res.json();

  if (!res.ok) throw new Error(data?.message || "Request failed");
  return data; // { success: true, message: "..." }
}

‚úÖ Practical UX tip

  • Show a loading state while the request runs ‚è≥
  • On success, show data.message ‚úÖ
  • On error, show a friendly error message üö´

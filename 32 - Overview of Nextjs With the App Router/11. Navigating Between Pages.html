1) Goal: Move between pages without full reloads üéØ

You already created routes like /, /cabins, /about, and /account. Now you need navigation links so users can move between pages.

What you want:

Navigation should feel like a Single Page Application (SPA) (fast transitions), even though Next.js can server-render pages.

You can add a normal anchor tag on the homepage to go to /cabins. It will navigate correctly, but it usually causes a full page reload.

Homepage using a normal anchor

  • üîÑ Full page reload (feels slower)
  • üì• Browser fetches the whole page again
  • üö´ Not the smooth SPA-like navigation you expect
Common beginner mistake:

3) Correct way: use Next.js Link ‚úÖ

Next.js provides a Link component that enables faster, client-side navigation. It feels like SPA navigation.

Import Link

Use Link (same prop name as <a href>)
<Link href="/cabins">Explore luxury cabins</Link>
Key UX improvement:

Clicking the link transitions immediately without a hard reload. The app now feels much more like an SPA.

Nice detail vs React Router:

React Router‚Äôs link uses to. Next.js uses href, matching native HTML.

6) Build a reusable Navigation component üß©

Instead of placing links manually on every page, create a reusable component.

Create a components folder (temporary approach)

  components/
    navigation.js
‚ö†Ô∏è Important nuance:

export default function Navigation() {
  return (
    <ul>
      <li><Link href="/">Home</Link></li>
      <li><Link href="/cabins">Cabins</Link></li>
      <li><Link href="/about">About</Link></li>
      <li><Link href="/account">Your account</Link></li>
    </ul>
  );
}
Use it in a page (example)

export default function Page() {
  return (
    <div>
      <Navigation />
      <h1>The Wild Oasis</h1>
      <p>Welcome to paradise.</p>
    </div>
  );
}
Result:

You can click between pages smoothly, with SPA-like navigation.

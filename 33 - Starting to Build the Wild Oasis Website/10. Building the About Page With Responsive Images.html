2) About Page: Replacing <img> with next/image üñºÔ∏è

The About page contains images that look responsive in the layout, but the raw images are large (e.g., ~2000px wide), which increases page weight. Next.js helps by optimizing images automatically.

Quick refresher: why use next/image?

  • üöÄ Serves correctly sized images (responsive srcset)
  • üß† Can output modern formats (like WebP) when appropriate
  • üß± Prevents layout shifts (when dimensions are known)
  • üõå Lazy loads images by default when off-screen

2.1 The ‚Äúeasy mode‚Äù: static image import ‚úÖ

If your image is local in /public (or available at build time), you can import it. Next.js can then infer the image dimensions automatically.

// About page (server component)
import Image from "next/image";

import about1 from "@/public/about-1.jpeg";
import about2 from "@/public/about-2.jpeg";

export default function Page() {
  return (
    <div>
      <Image src={about1} alt="About photo 1" placeholder="blur" />
      <Image src={about2} alt="About photo 2" placeholder="blur" />
    </div>
  );
}
Why this works well üëç
  • No need to manually set width and height (Next can infer them).
  • You can use placeholder="blur" (blur-up preview) for nicer UX.

2.2 The ‚Äúreal-world‚Äù problem: you only have a URL ‚ö†Ô∏è

Often, images come from a database (e.g., Supabase) as URLs. In that case, you may not know the image dimensions at build time. If you use:

<Image src="/about-1.jpeg" alt="..." />

Next.js will typically require you to provide width and height (or use another strategy) to prevent layout shifts.

3) Key Technique: Responsive Image without Fixed Dimensions (Using fill) üß±

When you want an image to fluidly fill a grid column and you don‚Äôt want fixed dimensions, use this pattern:

Step 1
Wrap the image in a parent container
Step 2
Make the parent position: relative
Step 3
Give the parent a size (often via aspect-ratio)
Step 4
Use fill + object-fit: cover on the Image
import Image from "next/image";

export default function AboutImageBlock() {
  return (
    <div className="relative aspect-square">
      <Image
        src="/about-1.jpeg"        // imagine this came from a DB URL
        alt="About photo"
        fill
        className="object-cover"
      />
    </div>
  );
}
What this achieves üß†
  • The parent defines the layout box (size + ratio).
  • The image ‚Äúfills‚Äù that box and stays responsive.
  • object-cover ensures the image crops nicely without distortion.

Useful variations

  • Stick the crop to the top (similar to object-position: top): className="object-cover object-top"
  • Video-like ratio (16:9): aspect-video
  • Square ratio (1:1): aspect-square
Common gotcha üöß

If the parent has no height, the image will appear ‚Äúbroken‚Äù (often height = 0). Using aspect-* or an explicit height fixes that.

4) ‚ÄúBlur placeholder‚Äù and ‚Äúquality‚Äù limitations (Important!) üß©

The Image component has a powerful API, but some options only work under certain conditions.

‚úÖ Works best with static imports

  • placeholder="blur" is straightforward for statically imported images.
  • quality={80} is easy to tune for background/large images.

‚ö†Ô∏è When you only have a URL

  • placeholder="blur" may not be available unless you also provide a blurDataURL (depends on how the image is provided).
  • Some optimizations are easiest when Next can analyze the image at build-time.

Practical takeaway: if the image comes from a DB, you often rely on the fill + parent container technique, and you may need extra steps for blur placeholders.

SEO note üîç

Reducing layout shift and improving image loading improves user experience and can contribute to better SEO signals. Next.js pushes you toward these best practices.

5) Mental Model: Static Import vs fill (When to Use Which) üß†

Option A ‚Äî Static import (preferred for local images)

  • Use when you have images in /public (or otherwise local/known at build time).
  • Easy dimensions + blur placeholder.

Option B ‚Äî URL + fill (common for DB-driven images)

  • Use when you only have a URL and want a fluid, responsive layout.
  • Requires a parent container that defines size (often with aspect-ratio).
  • Use object-cover to avoid distortion.

üß© Big picture: what you‚Äôre doing

In a Next.js project created with create-next-app, Tailwind can be installed automatically. But even if Tailwind is installed, you still need to import the generated CSS file so Tailwind‚Äôs styles become active.

‚úÖ Key idea

Tailwind becomes ‚Äúactive‚Äù when your app loads the file that contains Tailwind‚Äôs directives (usually globals.css).

What you already have (from create-next-app)

Tailwind is installed + config files exist

  • tailwind.config.js exists
  • globals.css contains Tailwind directives
  • You can style via utility classes like bg-blue-800, min-h-screen, etc.

‚ö° Step 1 ‚Äî Activate Tailwind (import globals.css)

Tailwind typically generates a global stylesheet that contains these 3 directives: 

@tailwind base;
@tailwind components;
@tailwind utilities;

To activate Tailwind, import that global CSS file somewhere that is guaranteed to load for the entire app. In the App Router, the best place is the root layout (app/layout.js).

‚ö†Ô∏è Common beginner confusion

  • ‚ÄúTailwind is installed but nothing changes‚Äù ‚Üí usually you forgot to import globals.css.
  • Custom CSS in globals.css can break things if it has invalid rules. Comment out suspicious blocks to isolate issues.

Example import (with alias)

If your project uses an import alias where @ maps to the project root (common in Next.js), the import might look like: 

// app/layout.js
import "@/app/_styles/globals.css";

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        {children}
      </body>
    </html>
  );
}

After this, you should immediately see default Tailwind base styling (and any Tailwind classes inside your components will start working).

üß† Step 2 ‚Äî Quick Tailwind mental model

Concept

Tailwind = utility classes

Instead of writing custom CSS selectors, you apply small ‚Äúutility‚Äù classes directly in JSX via className.

Why people like it

Fast styling + consistent scale

Tailwind uses a predefined spacing and size scale. Example: gap-4 corresponds to 1rem (16px).

Example: styling the body

// app/layout.js
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body className="min-h-screen bg-blue-800 text-gray-50">
        {children}
      </body>
    </html>
  );
}

Notes:

  • bg-blue-800 sets background color.
  • text-gray-50 sets text color.
  • min-h-screen ensures the body covers the full viewport height.

üß© Step 3 ‚Äî VS Code IntelliSense (autocomplete)

Tailwind class autocomplete is not built-in by default. It usually comes from a VS Code extension.

‚úÖ Recommended

Install the Tailwind CSS IntelliSense extension so you get:

  • Autocomplete for classes (e.g., typing bg- shows options)
  • Hover previews that show generated CSS
  • Linting for invalid class names

Tip: This is optional, but it reduces mistakes and speeds up styling.

üéõÔ∏è Step 4 ‚Äî Customize theme colors in tailwind.config.js

Tailwind ships with default colors (blue, gray, etc.). For a real project, you often want brand colors. The lecture uses a colors.json from starter files, then copies those values into tailwind.config.js under theme.extend.

What ‚Äúextend‚Äù means

Extend adds new tokens (like primary, accent) while keeping Tailwind defaults. You avoid replacing Tailwind‚Äôs entire color palette unless you really want to.

Example config snippet

  content: [
    "./app/**/*.{js,ts,jsx,tsx}",
    "./components/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {
      colors: {
        primary: {
          50: "#eef2ff",
          100: "#e0e7ff",
          200: "#c7d2fe",
          300: "#a5b4fc",
          400: "#818cf8",
          500: "#6366f1",
          600: "#4f46e5",
          700: "#4338ca",
          800: "#3730a3",
          900: "#312e81",
          950: "#1e1b4b"
        },
        accent: {
          50: "#fff7ed",
          100: "#ffedd5",
          200: "#fed7aa",
          300: "#fdba74",
          400: "#fb923c",

Using your new colors

  {children}
</body>

‚ö†Ô∏è Debug note (realistic dev workflow)

If you paste theme config and suddenly get weird errors: don‚Äôt panic. Comment out suspicious CSS blocks (e.g., in globals.css) and re-check your object braces in tailwind.config.js. Small syntax mistakes can cause confusing editor errors.

üß™ Alternatives: CSS Modules, styled-components

Option What it is When it‚Äôs a good fit
Tailwind CSS Utility-first classes inside className Fast UI iteration, consistent spacing/colors, team conventions
CSS Modules Scoped CSS files (e.g., Button.module.css) Prefer ‚Äúclassic CSS‚Äù, want automatic scoping per component
styled-components CSS-in-JS with styled primitives You already have a mature styled-components codebase and patterns

In the course, Tailwind is chosen because it‚Äôs popular, integrates easily with Next.js, and the project provides most styles already.

1) ‚úÖ What is Prefetching?

Prefetching means fetching data before the user actually needs it on the UI. Think: ‚ÄúAdvance loading.‚Äù

üí° Simple idea: If the user is on page 7, we can already fetch page 8 in the background and store it in the cache. Then when they click ‚ÄúNext‚Äù, the UI can render instantly from cache.

In pagination, the most common prefetch targets are:

  • ‚û°Ô∏è Next page (page + 1)
  • ‚¨ÖÔ∏è Previous page (page ‚àí 1)

2) üéØ Why it matters for Pagination

Without prefetching, when the user clicks Next, the app must: (1) request data ‚Üí (2) wait ‚Üí (3) show spinner ‚Üí (4) render.

üò¨ Without Prefetching

  • User clicks next page
  • React Query fetches on-demand
  • Spinner appears
  • Data renders after network delay

üòé With Prefetching

  • While on current page, app preloads nearby pages
  • When user navigates, data is already in cache
  • Minimal to no spinner
  • Feels ‚Äúinstant‚Äù

‚ö° Result: The app feels fast even if pagination is truly server-side, because React Query serves cached pages immediately.

3) üß© How to Implement with React Query

You implement prefetching where you already fetch the paginated data ‚Äî commonly inside a hook like useBookings().

Step A ‚Äî Get the Query Client

React Query provides useQueryClient() so we can manually prefetch. 

import { useQuery, useQueryClient } from "@tanstack/react-query";

const queryClient = useQueryClient();

Step B ‚Äî Prefetch the next page

The prefetch call looks like useQuery: it needs a queryKey and a queryFn. 

// Example variables (these typically come from URL search params)
// page, filter, sortBy, etc.

queryClient.prefetchQuery({
  queryKey: ["bookings", filter, sortBy, page + 1],
  queryFn: () => getBookings({ filter, sortBy, page: page + 1 }),
});

üß† Key point: Prefetch uses the same key structure as the real query. That‚Äôs how React Query knows ‚Äúthis cached data belongs to page 8 (not page 7).‚Äù

Step C ‚Äî Prefetch the previous page

queryClient.prefetchQuery({
  queryKey: ["bookings", filter, sortBy, page - 1],
  queryFn: () => getBookings({ filter, sortBy, page: page - 1 }),
});

4) üõ°Ô∏è Important Guards (Avoid useless fetches)

You should only prefetch pages that actually exist. Otherwise, you might fetch ‚Äúpage 999‚Äù which returns empty data and wastes requests.

Compute pageCount

If you have count (total results) and PAGE_SIZE, then: 

const pageCount = Math.ceil(count / PAGE_SIZE);

Guard conditions

// Prefetch next page only if not on the last page
if (page < pageCount) {
  queryClient.prefetchQuery({
    queryKey: ["bookings", filter, sortBy, page + 1],
    queryFn: () => getBookings({ filter, sortBy, page: page + 1 }),
  });
}

// Prefetch previous page only if not on the first page
if (page > 1) {
  queryClient.prefetchQuery({
    queryKey: ["bookings", filter, sortBy, page - 1],
    queryFn: () => getBookings({ filter, sortBy, page: page - 1 }),
  });
}

‚úÖ Outcome: The user can move back/forward without seeing spinners (most of the time), because nearby pages are already cached.

5) üß† Mental Model: Cache + Query Keys

React Query caches results by queryKey. You can treat the query key like a ‚Äúcache address.‚Äù

  • Same key ‚Üí React Query reuses cached data
  • Different key ‚Üí React Query stores a separate cache entry

Example key structure

We include every ‚Äústate‚Äù that changes the data:

  • Resource name: "bookings"
  • Filter object: filter
  • Sort object: sortBy
  • Page number: page

So the key becomes: ["bookings", filter, sortBy, page]

üîé Tip: Open React Query Devtools to visually see how many cache entries you‚Äôre creating as you change filter/sort/page.

6) üîÅ Alternative: Infinite Queries (Infinite Scroll)

Instead of classic pagination + prefetching, you can implement infinite scrolling using React Query‚Äôs useInfiniteQuery.

  • ‚úÖ Great for feeds (social apps, timelines)
  • ‚úÖ User keeps scrolling; you load the next chunk automatically
  • ‚ö†Ô∏è Not always best for admin tables (where jumping to page 12 matters)

üìå Practical takeaway: Prefetching is one of the best improvements for classic pagination UX.

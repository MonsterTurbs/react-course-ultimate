5) Quick project tour: what matters most üóÇÔ∏è

You will see many files. You do not need to understand all of them immediately. Focus on the ones that explain how Next.js works.

Key files/folders (mentioned):
  • package.json ‚Äî a Next.js app is also a Node.js app
  • next.config.js ‚Äî config file (starts empty, later you‚Äôll add settings)
  • public/ ‚Äî static assets (like in Vite / CRA)
Most important:
  • app/ folder ‚Äî enables the App Router ‚úÖ
  • Inside app/ you‚Äôll see conventions like page.js and layout.js
App Router vs Pages Router (practical rule):

App Router uses the app/ folder. If you were using the legacy Pages Router, you‚Äôd use a pages/ folder instead.

8) Verify SSR: check ‚ÄúView Page Source‚Äù üïµÔ∏è‚Äç‚ôÇÔ∏è

The lecture emphasizes a simple proof that SSR is happening: use View Page Source in the browser.

Why ‚ÄúView Page Source‚Äù matters:
  • Elements tab shows the current DOM after JavaScript runs.
  • View Page Source shows what was downloaded from the server.
SSR confirmation:

If you see your <h1>Hello Next!</h1> in the page source, the server generated that HTML and sent it.

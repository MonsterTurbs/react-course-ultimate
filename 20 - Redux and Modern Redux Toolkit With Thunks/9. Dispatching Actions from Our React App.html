üéØ Big Goal of the Lecture

Learn how to dispatch actions to the Redux store from inside React components. In short: instead of calling store.dispatch(...) directly, we use React-Redux hooks.

‚úÖ Mental Model (Simple)

  • UI event happens (click / submit) üñ±Ô∏è
  • Component calls dispatch(actionCreator(payload)) üì¶
  • Reducer updates store state üßÆ
  • Components using useSelector re-render automatically üîÅ

üß© Key Hooks You Use

Hook Purpose How to think about it
useDispatch() Gives you the dispatch function ‚ÄúGet the remote control to send actions to the store.‚Äù üì°
useSelector(selectorFn) Reads state from the store ‚ÄúSubscribe this component to a slice of state.‚Äù üëÄ

üîé Rule of thumb: Use useSelector to read data. Use useDispatch inside event handlers to update data.

üë§ Part 1 ‚Äî Creating a Customer (CreateCustomer Component)

What the component already has

  • Two controlled inputs: fullName and nationalId ‚úçÔ∏è
  • A button that triggers handleClick() üß∑

What you implement

  • Get dispatch via useDispatch()
  • Import the action creator createCustomer from customerSlice
  • Guard clause: if inputs are empty, do nothing
  • Dispatch action: dispatch(createCustomer(fullName, nationalId))
// CreateCustomer.jsx (core idea)
import { useDispatch } from "react-redux";
import { createCustomer } from "./customerSlice";

function CreateCustomer() {
  const dispatch = useDispatch();

  function handleClick() {
    if (!fullName || !nationalId) return; // ‚úÖ guard clause

    dispatch(createCustomer(fullName, nationalId));
  }

  // ...render inputs + button
}

Why action creators matter üß†

Action creators reduce typos and keep action structure consistent. Instead of manually writing { type: "customer/createCustomer", payload: ... }, you just call a function.

What happens after dispatch

  • The Redux store updates the customer slice ‚úÖ
  • Any component that selects customer.fullName will re-render automatically üîÅ

üß≠ Part 2 ‚Äî Hide/Show UI Based on Store State (App.js)

After creating a customer, the ‚ÄúCreate Customer‚Äù form is no longer needed. So we conditionally render based on customer.fullName. 

// App.js (core idea)
import { useSelector } from "react-redux";

function App() {
  const fullName = useSelector((state) => state.customer.fullName);

  return (
    <div>
      {fullName === "" ? (
        <CreateCustomer />
      ) : (
        <>
          <Customer />
          <BalanceDisplay />
          <AccountOperations />
        </>
      )}
    </div>
  );
}

‚úÖ Beginner tip: Keep selectors small. If you only need fullName, select only that. Smaller selections often mean simpler re-render behavior.

üè¶ Part 3 ‚Äî Dispatching Account Actions (AccountOperations Component)

What this UI represents

  • Deposit
  • Withdraw
  • Request loan (amount + purpose)
  • Pay loan

Step-by-step pattern (repeat for each button)

  1. Get dispatch using useDispatch()
  2. Import the correct action creator from accountSlice
  3. Guard clause: if required fields are missing, return
  4. dispatch(actionCreator(payload))
  5. Reset local input state (optional but recommended)
// AccountOperations.jsx (dispatch + guard clauses)
import { useDispatch, useSelector } from "react-redux";
import { deposit, withdraw, requestLoan, payLoan } from "./accountSlice";

function AccountOperations() {
  const dispatch = useDispatch();

  // ‚úÖ Read account data once, then destructure
  const account = useSelector((state) => state.account);
  const {
    loan: currentLoan,
    loanPurpose: currentLoanPurpose,
    balance,
  } = account;

  function handleDeposit() {
    if (!depositAmount) return;
    dispatch(deposit(depositAmount));
    setDepositAmount("");
  }

  function handleWithdraw() {
    if (!withdrawalAmount) return;
    dispatch(withdraw(withdrawalAmount));
    setWithdrawalAmount("");
  }

  function handleRequestLoan() {
    if (!loanAmount || !loanPurpose) return;
    dispatch(requestLoan(loanAmount, loanPurpose));
    setLoanAmount("");
    setLoanPurpose("");
  }

  function handlePayLoan() {
    dispatch(payLoan());
  }

  // ...render buttons + current loan UI
}

üß† Why useSelector here too?

You need account data to display things like current loan and balance. Once you dispatch, Redux updates state and this component will re-render with the new values.

Display loan only when it exists

{currentLoan > 0 && (
  <p>
    Current loan: {currentLoan} ({currentLoanPurpose})
  </p>
)}

üßæ Practical note: Loan UI is conditional because paying the loan resets loan values (loan goes back to 0 and purpose becomes empty).

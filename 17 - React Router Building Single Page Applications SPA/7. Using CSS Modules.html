üß† Quick concept: What is a CSS Module?

A CSS Module is a regular CSS file that becomes component-scoped. That means the classes you define in that file are meant to be used only by the component that imports them.

Key benefit: No more naming clashes. Two different components can both have a class named nav, and they won‚Äôt conflict.

üòÑ Good news: CSS Modules work out of the box in Create React App and Vite. Nothing to install.

üóÇÔ∏è Step 1 ‚Äî Create the module file

The naming convention is critical. For a component named PageNav, the CSS module should be:

PageNav.module.css

Example (simple placeholder styles):

/* PageNav.module.css */
.nav {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/* ‚úÖ Scoped descendant selector: affects only ul INSIDE this component's .nav */
.nav ul {
  list-style: none;
  display: flex;
  gap: 12px;
  padding: 0;
  margin: 0;
}

‚ö†Ô∏è Avoid writing a plain element selector like ul { ... } inside a module. Even though it‚Äôs in a module file, it can still become effectively global and defeat the purpose.

üîå Step 2 ‚Äî Import and use module classes in JSX

CSS Modules export all classes as one JavaScript object (commonly named styles). 

// PageNav.jsx
import { NavLink } from "react-router-dom";
import styles from "./PageNav.module.css";

export default function PageNav() {
  return (
    <nav className={styles.nav}>
      <ul>
        <li><NavLink to="/">Home</NavLink></li>
        <li><NavLink to="/pricing">Pricing</NavLink></li>
        <li><NavLink to="/product">Product</NavLink></li>
      </ul>
    </nav>
  );
}

üìå Important: In JSX, module classNames are not strings. You use JavaScript expression mode:

className={styles.nav} (not className="nav")

üîê Why CSS Modules prevent naming clashes

When your app runs, CSS Modules transform your class names into unique names (they ‚Äúhash‚Äù them). Example:

  • .nav becomes something like nav__a1b2c behind the scenes.
  • Another module‚Äôs .nav becomes nav__x9y8z ‚Äî different, so no collision.

‚úÖ Result: You can reuse class names like container, title, nav across components safely.

üß™ Demo idea: Two components, same class name

You can create an AppNav component with its own module and also define .nav there. They won‚Äôt clash. 

/* AppNav.module.css */
.nav {
  background: rebeccapurple;
  color: white;
  padding: 8px 12px;
  border-radius: 10px;
}

// AppNav.jsx

export default function AppNav() {
  return (
    <nav className={styles.nav}>
      App Navigation
    </nav>
  );
}

‚ö†Ô∏è You might see some devs do: const { nav } = styles and then className={nav}. It works, but it can reduce clarity in large JSX files. Using styles.nav is often easier to scan.

üåç Global CSS is still allowed (and sometimes required)

Some CSS should be global, like:

  • CSS reset (box-sizing, margins)
  • CSS variables (:root { --color: ... })
  • Base typography on body

In a Vite project, you can keep a global index.css and import it in main.jsx: 

// main.jsx
import React from "react";
import ReactDOM from "react-dom/client";
import App from "./App.jsx";
import "./index.css";

ReactDOM.createRoot(document.getElementById("root")).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);

‚úÖ Best practice: Keep global CSS minimal (variables + reset). Put component-specific styles in CSS modules.

üß∑ Using :global() inside a module

Sometimes you need to style a class name that does not come from your module ‚Äî for example, a class provided by a library. In React Router, NavLink automatically adds an active class to the current route.

‚ö†Ô∏è If you write .nav .active { ... } inside a module, it may not work as expected because active can get treated like a module class.

‚úÖ Correct approach: mark the external class as global.

/* PageNav.module.css */
.nav :global(.active) {
  background: #16a34a; /* green */
  color: white;
  padding: 4px 10px;
  border-radius: 999px;
}

üß† Rule of thumb: Use :global() when the class name originates outside your module (router, third-party libs, etc.).

üõ†Ô∏è Common mistakes and fixes

1) ‚ÄúMy styles don‚Äôt apply‚Äù

  • Check the filename ends with .module.css.
  • Make sure you imported it: import styles from "./X.module.css".
  • Make sure you used className={styles.someClass} (not a plain string).

2) ‚ÄúIt styles the whole app (I thought it was scoped)‚Äù

  • Avoid naked element selectors like ul { ... }.
  • Prefer scoped selectors like .nav ul { ... }.

3) ‚ÄúNavLink active styling doesn‚Äôt work‚Äù

  • Use :global(.active) inside your module selector chain.

‚úÖ What you do

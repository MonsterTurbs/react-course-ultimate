1) The Problem We‚Äôre Solving üß†

You have a List component that already contains: stateful logic (collapse/show fewer items) and UI (title + buttons + layout).

You want to reuse that same List component for different item types: products and companies.

Because you don‚Äôt only need to ‚Äúpass JSX content‚Äù. You need to pass instructions on how to render each item (and you need access to the list‚Äôs internal displayItems logic).

2) Render Props in One Sentence ‚úÖ

Render props means you pass a prop (commonly named render) that is a function. The component calls that function to know what to render and how to render it.

Key idea: ‚ÄúInversion of Control‚Äù üîÅ

The List component no longer decides which component to render for each item. The user of the List decides by passing the render function.

3) How It Works (Step-by-Step) üõ†Ô∏è

  1. Pass a render prop into <List />.
    This function describes what to render for each item.
  2. Receive render inside List (alongside title and items).
  3. Call render(item) inside .map() instead of hardcoding a component.
  4. Reuse the same List for different item types by changing only the render function.

4) Minimal Example Code üßæ

This mirrors the lecture idea (products vs companies) but keeps it beginner-friendly.

4.1 List component (generic) üì¶

function List({ title, items, render }) {
  const [isCollapsed, setIsCollapsed] = React.useState(false);

  // Example logic: show only first 3 when collapsed
  const displayItems = isCollapsed ? items.slice(0, 3) : items;

  return (
    <div>
      <h2>{title}</h2>

      <ul>
        {displayItems.map((item) => (
          <li key={item.id}>
            {render(item)}
          </li>
        ))}
      </ul>

      <button onClick={() => setIsCollapsed((c) => !c)}>
        {isCollapsed ? "Show all" : "Show less"}
      </button>
    </div>
  );
}

What changed? ‚úÖ

Only the render function. Everything else (collapse logic, buttons, layout) stays shared.

5) Why the Render Prop Must Be a Function üß†

If you passed only a component (like <CompanyItem />) without a function, the List would have no idea:

  • Which item data to pass into that component
  • How to connect the item from displayItems to the UI element
  • How to produce one UI element per item

The function form (item) => <Component item={item} /> solves this by providing a clear mapping.

6) When to Use Render Props (Practical Rule) üìå

Use it when ‚úÖ
  • You must reuse logic + UI together
  • The parent needs control over how items render
  • children is too limited because you need ‚Äúrender instructions‚Äù
Prefer custom hooks when ‚úÖ
  • You only need to reuse stateful logic (no shared layout)
  • You want the UI to be fully custom per feature

7) Common Pitfalls & Tips ‚ö†Ô∏è

  • Key props: Your list items still need stable key values.
  • Naming: render is common, but you might also see children used as a function (‚Äúfunction as children‚Äù).
  • Don‚Äôt overuse: Hooks handle many sharing cases today. Render props shine when UI + logic must be reused together.

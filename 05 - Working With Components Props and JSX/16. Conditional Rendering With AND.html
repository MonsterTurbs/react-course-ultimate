üß† What conditional rendering means

Conditional rendering means rendering a piece of UI (a JSX chunk or an entire component) only when a condition is met.

In this lecture‚Äôs example üçï

Render an ‚ÄúOrder‚Äù box in the footer only if the restaurant is currently open (isOpen is true).

üîÄ Using && with short-circuiting

The AND operator (&&) can be used to conditionally render JSX because of short-circuiting.

Short-circuit rule ‚úÖ

If the left side is truthy, JavaScript returns the right side. If the left side is falsey, it returns the left side (and skips evaluating the right side).

Small example (render text) üßæ

function Footer() {
  const hour = new Date().getHours();
  const openHour = 12;
  const closeHour = 22;
  const isOpen = hour >= openHour && hour <= closeHour;

  return (
    <footer>
      {isOpen && "Open"}
    </footer>
  );
}

What happens ‚úÖ

  • If isOpen is true ‚Üí React receives "Open" and renders it.
  • If isOpen is false ‚Üí React receives false and renders nothing.

üö´ Why React doesn‚Äôt render booleans

React will not render true or false into the DOM. That‚Äôs why patterns like {condition && <JSX />} work nicely. 

<div>{false}</div>  // renders nothing
<div>{true}</div>   // renders nothing

Key idea ‚úÖ

If the expression results in a boolean, React ignores it. If it results in JSX/text/number, React renders it.

üßæ Real use case: show ‚ÄúOrder‚Äù box only when open

Instead of just rendering the word ‚ÄúOpen,‚Äù you can render a whole JSX block. This follows the JSX rule you learned earlier: JSX is an expression, so it can appear inside { }. 

function Footer() {
  const hour = new Date().getHours();
  const openHour = 12;
  const closeHour = 22;
  const isOpen = hour >= openHour && hour <= closeHour;

  return (
    <footer className="footer">
      {isOpen && (
        <div className="order">
          <p>
            We're open until {closeHour}:00. Come visit us or order online.
          </p>
          <button className="btn">Order</button>
        </div>
      )}
    </footer>
  );
}

Result ‚úÖ

If open ‚Üí show message + button. If closed ‚Üí show nothing (or you could render a closed message using a different technique).

üåó Truthy/falsey values: arrays and length

The && pattern also works with truthy/falsey values (not just booleans). But this is where beginners get surprised.

Problem: empty arrays are truthy üòÖ

const pizzas = []; // still truthy

return (
  <ul>
    {pizzas && pizzas.map(p => <Pizza key={p.name} pizzaObj={p} />)}
  </ul>
);

// Result: the <ul> still renders (it's just empty).

Fix ‚úÖ

Check the length instead: pizzas.length > 0

const pizzas = []; // none

return (
  <div>
    {pizzas.length > 0 && (
      <ul className="pizzas">
        {pizzas.map(p => <Pizza key={p.name} pizzaObj={p} />)}
      </ul>
    )}
  </div>
);

// Now: if there are no pizzas, the UL isn't rendered at all.

0Ô∏è‚É£ Common bug: why 0 appears in the UI

A classic issue happens when you do this: 

{pizzas.length && <PizzaList />}

When pizzas.length is 0, the expression becomes: 0 && .... JavaScript returns 0 (the left side), and React will actually render 0.

Why it happens ‚ö†Ô∏è

React ignores booleans, but it will render numbers. So 0 shows up.

Correct pattern ‚úÖ (force a boolean condition)

{pizzas.length > 0 && <PizzaList />}

Rule ‚úÖ

Make the left side clearly boolean (condition or length > 0), not a number.

üß≠ Guidelines + when to prefer ternary

When && is great ‚úÖ

  • You only need to render something in the true case
  • You want a short, readable pattern
  • The left side is a clear boolean condition

When to avoid && ‚ö†Ô∏è

  • The left side might be 0 or another value that React will render
  • You want an else UI (true vs false UI)
  • The expression becomes hard to read (nested conditions)

Reviewer / Study Guide ‚Ä¢ Focus: rendering SVG stars, click-to-rate state, and ‚Äúfilled vs empty‚Äù stars using conditional rendering.

Goal üéØ

Create a dynamic star rating UI that updates when a user clicks a star. Then display filled stars up to the rating and empty stars after it.

Core Ideas üß†
  • State (useState) triggers re-render when rating changes.
  • Child-to-parent communication: parent owns state; child triggers updates via a handler prop.
  • Conditional rendering: choose full vs empty star based on a boolean.
  • Index math: array indices start at 0, rating starts at 1.

1) Why create a Star component? üß©

The star SVG JSX is a bit ‚Äúheavy.‚Äù Putting it in a separate component makes your code easier to read and reuse. It also helps isolate the logic for styles and click handling.

What the Star component will do

  • Render a star icon (SVG) inside a clickable element.
  • Accept a handler prop (e.g., onRate) to notify the parent when clicked.
  • Accept a boolean prop (e.g., full) to decide ‚Äúfilled vs empty.‚Äù

2) Make stars visible: styling the SVG ‚ú®

If your SVG appears ‚Äúinvisible,‚Äù it‚Äôs often because of missing dimensions. A simple fix is to wrap the SVG in a <span> and apply width/height.

What you set Why it matters
width, height Ensures the icon occupies space; otherwise it can render with 0 height visually.
display: block Removes inline spacing quirks and makes sizing more predictable.
cursor: pointer Gives user feedback that the element is clickable.
role="button" Accessibility hint for screen readers (important when using non-button elements).

3) Clicking a star updates UI: you need state üîÅ

If the UI must change after an event (click), React needs to re-render ‚Äî and that strongly suggests state. Here, the state is simply the current rating.

Rule of thumb ‚úÖ If something changes on screen because of user interaction, you usually need state for it.

4) Pattern: parent owns state, child triggers update üì§

The rating state lives in StarRating. Each Star is a child. The child calls a function passed via props to update the rating. This is the standard React pattern used across many apps.

5) Full vs empty stars: boolean + conditional rendering ‚≠ê/‚òÜ

You decide whether a star is ‚Äúfull‚Äù by comparing the current rating with the star‚Äôs position. For star number i + 1, it is full if: rating >= i + 1.

Key formula üßÆ full = rating >= i + 1

Because indices start at 0, but ratings start at 1.

import { useState } from "react";

// Styles defined outside so they don't re-create on each re-render (minor optimization)
const containerStyle = {
  display: "flex",
  alignItems: "center",
  gap: "16px",
};

const starContainerStyle = {
  display: "flex",
  gap: "0px", // lecture removes gap for tighter stars
};

const textStyle = {
  lineHeight: "1",
  margin: "0",
};

const starStyle = {
  width: "48px",
  height: "48px",
  display: "block",
  cursor: "pointer",
};

export default function StarRating({ maxRating = 5 }) {
  const [rating, setRating] = useState(0);

  function handleRating(newRating) {
    setRating(newRating);
  }

  return (
    <div style={containerStyle}>
      <div style={starContainerStyle}>
        {Array.from({ length: maxRating }, (_, i) => (
          <Star
            key={i}
            full={rating >= i + 1}
            onRate={() => handleRating(i + 1)}
          />
        ))}
      </div>

      {/* show empty when rating is 0 */}
      <p style={textStyle}>{rating || ""}</p>
    </div>
  );
}

function Star({ full, onRate }) {
  return (
    <span role="button" onClick={onRate} style={starStyle}>
      {full ? <FullStar /> : <EmptyStar />}
    </span>
  );
}

Mini Walkthrough (what happens when you click) ü™ú

  1. You render maxRating stars using Array.from({ length: maxRating }).
  2. Each star gets an onRate callback that sets rating to i + 1.
  3. When clicked, React updates state (setRating) ‚Üí triggers re-render.
  4. During re-render, each star recalculates full using rating >= i + 1.
  5. Result: stars up to rating are full; the rest are empty.

Common mistakes & fixes üõ†Ô∏è

  • Problem: Stars don‚Äôt show (looks like empty space).
    Fix: Give the SVG (or wrapper) explicit width/height and use display: block.
  • Problem: Rating shows 0 but you want ‚Äúblank‚Äù.
    Fix: Render {rating || ""} (short-circuit).
  • Problem: Using index as key feels ‚Äúnot ideal‚Äù.
    Fix: For a fixed static list of stars it‚Äôs acceptable; for dynamic lists, prefer stable IDs.
  • Problem: Confusing prop name onClick (native) vs custom handler.
    Fix: Use a semantic name like onRate to make intent obvious.

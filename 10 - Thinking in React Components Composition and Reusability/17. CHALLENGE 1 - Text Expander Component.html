üéØ What you are building

A Text Expander is a reusable UI component that shows a preview of a long text (e.g., first 10‚Äì20 words), plus a button like Show more. When the button is clicked, the component expands to show the full text, and the button label switches to Show less.

‚úÖ Key behaviors

  • Collapsed view shows only the first N words + ‚Äú...‚Äù
  • Expanded view shows the entire text
  • Button toggles expanded ‚Üî collapsed
  • Button label changes depending on state
  • Reusable via props (public API)

üß† Core React ideas used in this challenge

1) children prop (slot content)

The text is provided between the component‚Äôs opening and closing tags: 

<TextExpander>
  This is the text that will be expanded/collapsed.
</TextExpander>

That text becomes available in the component as children. This is a classic example of component composition.

2) State for toggling

When the UI needs to change due to user interaction, you need state. Here, the state is typically a boolean: 

const [isExpanded, setIsExpanded] = useState(false);

3) Public API via props

This is a reusable component, so it should allow configuration using props such as: collapsedNumWords, buttonColor, expandButtonText, collapseButtonText, expanded (default open), and className.

üí° Think like this: you are the creator of the component, and someone else is the consumer. Your props are the component‚Äôs ‚Äúpublic API‚Äù.

Step B ‚Äî Initialize state from a prop (seed state)

If the consumer wants the expander open by default, use expanded as the initial state value:

üß† This is fine because expanded is only used as initial seed data. You are not trying to keep the state synchronized with prop changes.

Step C ‚Äî Compute the display text

When collapsed, show only the first N words. When expanded, show all content.

  ? children
  : `${String(children)
      .split(" ")
      .slice(0, collapsedNumWords)
      .join(" ")}...`;
‚ö†Ô∏è Note: children can be more than a plain string in React. For this challenge, it is expected to be text. Using String(children) helps avoid crashes if something non-string is passed.

Step D ‚Äî Toggle on click

  setIsExpanded((exp) => !exp);
}

‚úÖ Complete reference solution (clean + reusable)

Below is a complete implementation that matches the challenge requirements and demonstrates a clear public API. 

import { useState } from "react";

export default function TextExpander({
  collapsedNumWords = 20,
  expandButtonText = "Show more",
  collapseButtonText = "Show less",
  buttonColor = "#1f09cd",
  expanded = false,
  className = "",
  children,
}) {
  const [isExpanded, setIsExpanded] = useState(expanded);

  const buttonStyle = {
    background: "none",
    border: "none",
    padding: 0,
    marginLeft: 6,
    cursor: "pointer",
    color: buttonColor,
    font: "inherit",
  };

  const displayText = isExpanded
    ? children
    : `${String(children)
        .split(" ")
        .slice(0, collapsedNumWords)
        .join(" ")}...`;

  function handleToggle() {
    setIsExpanded((exp) => !exp);
  }

  return (
    <div className={className}>
      <span>{displayText}</span>
      <button style={buttonStyle} onClick={handleToggle}>
        {isExpanded ? collapseButtonText : expandButtonText}
      </button>
    </div>
  );
}
‚úÖ Pattern used: state toggle + derived display text + props as API

üß™ Example usages (how the consumer uses the API)

These examples show how the same component can look different with different props.

  Space travel is the ultimate adventure. Imagine soaring past the stars...
</TextExpander>

// Custom button labels + fewer collapsed words + custom color

  expandButtonText="Show text"
  collapseButtonText="Hide text"
  buttonColor="#e11d48"

  Space travel is the ultimate adventure. Imagine soaring past the stars...
</TextExpander>

// Expanded by default + custom container styling via className
<TextExpander expanded={true} className="box">
  Space travel is the ultimate adventure. Imagine soaring past the stars...
</TextExpander>
üí° Why className? Sometimes consumers want styling control from their app CSS. Allowing className gives them a hook without forcing your component to depend on external styles.

‚ö†Ô∏è Common pitfalls and how to avoid them

  • Forgetting to import useState: ensure import { useState } from "react";.
  • Using the wrong initial prop name: in the transcript, ‚Äúextended‚Äù was mentioned as a mistake. Use expanded.
  • Off-by-one words: verify slice(0, collapsedNumWords) matches your expectation.
  • children not being a string: this challenge expects text, but guard with String(children) for safety.
  • Too many props: keep the API balanced‚Äîuse defaults so the consumer doesn‚Äôt need to pass everything.
üßØ If you allow consumers to pass many props, always provide defaults. Otherwise the component becomes hard to use.

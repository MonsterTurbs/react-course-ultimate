1) Two ways to do component composition

In React, you can compose components (pass JSX into a component) in two common ways:

  • ‚úÖ Implicit slot using children (most common)
  • ‚úÖ Explicit slot using a named prop like element, header, footer, etc.

Key idea

Both approaches pass JSX into a component. They differ mainly in API design and readability.

2) The usual approach: children (implicit)

The children prop represents whatever you place between a component‚Äôs opening and closing tags: 

// Box.jsx
function Box({ children }) {
  return <div className="box">{children}</div>;
}

// Usage
<Box>
  <MovieList movies={movies} />
</Box>

‚úÖ Why this is preferred

  • Reads like HTML nesting (very natural)
  • Common pattern across the React ecosystem
  • Simple when there is one main ‚Äúslot‚Äù

3) Alternative approach: explicit prop (e.g., element)

Instead of children, you can accept a named prop such as element (or any name: content, left, right, etc.).

Box accepts element prop

// Box.jsx (alternative)
function Box({ element }) {
  return <div className="box">{element}</div>;
}

Usage: pass JSX into the element prop

// App.jsx
<Box element={<MovieList movies={movies} />} />

‚úÖ Outcome

The app looks the same. You simply passed the JSX explicitly via a prop instead of implicitly via children.

4) Passing multiple elements via explicit prop

If you want to pass multiple elements via an explicit prop, you often need a fragment: 

<Box
  element={
    <>
      <Search />
      <NumResults movies={movies} />
    </>
  }
/>

üß† Why fragment is needed

JSX expressions in a prop must evaluate to a single value. A fragment groups multiple elements into one.

5) When the explicit prop approach makes sense

Using an explicit prop can be a viable pattern when you want multiple named slots. Example: a component might accept header, content, and footer.

Good use case

  • You want to pass multiple pieces of JSX and name them clearly.
  • Named slots are easier to understand than nesting everything in children.

The lecture notes that React Router uses a similar pattern in some APIs, which is why it‚Äôs useful to recognize.

6) Why the lecture returns to children

For the Box use case in this app, children is cleaner and more idiomatic:

  • Better readability in the JSX layout
  • Less noise than putting large JSX inside a prop assignment
  • Matches how React code is commonly written

‚úÖ Practical guidance

Prefer children for the ‚Äúmain slot‚Äù most of the time. Use explicit props when you truly need multiple named slots.

7) VS Code tip: jump to definition

Handy productivity trick in VS Code:

  • Hover over a component name
  • Hold Ctrl (Windows) / Cmd (Mac)
  • Click to jump to the component definition

Why this matters

It speeds up reading and understanding codebases‚Äîespecially when working with many components.

üö´ Why Regular <a href> Links Are Wrong in an SPA

When you use an anchor tag like this:

<a href="/bookings">Bookings</a>

It triggers a full page reload. Symptoms:

  • Noticeable ‚Äúflash‚Äù as the browser reloads
  • App state can reset unexpectedly
  • Feels slower and not like a SPA
‚ùå In a React Router app, prefer Link or NavLink so navigation stays client-side.

‚úÖ Fix: Use NavLink for Smooth Navigation + Active State

React Router‚Äôs NavLink does two things you want for menus:

  • Client-side navigation (no reload)
  • Automatically adds an active class (or provides a way to style active state)

Minimal Example

import { NavLink } from "react-router-dom";

function MainNav() {
  return (
    <nav>
      <ul>
        <li>
          <NavLink to="/dashboard">Home</NavLink>
        </li>
        <li>
          <NavLink to="/bookings">Bookings</NavLink>
        </li>
      </ul>
    </nav>
  );
}
üí° Why NavLink (not just Link)? Because menus typically need an active highlight.

üé® Styling a 3rd‚ÄëParty Component with styled-components

Key trick from the lecture:

  • You can style not only HTML tags (styled.a, styled.ul)
  • But also React components from other libraries by passing them in: styled(NavLink)

Pattern

import styled from "styled-components";
import { NavLink } from "react-router-dom";

const StyledNavLink = styled(NavLink)`
  /* your link styles */

  &.active {
    /* styles when the link matches the current route */
  }
`;

Important detail: React Router applies active on the link that matches the current URL, so you can target it with:

&.active { /* active link styles */ }
‚ö†Ô∏è Common confusion: The & refers to the component itself (the rendered anchor). It‚Äôs like writing: a.active { ... } but scoped.

Common Pitfalls

  • Logo not found: images not moved to public/
  • Page flashes/reloads: still using <a href> instead of NavLink
  • Active styling not working: not targeting &.active in styled component
  • Icons not showing: wrong import path (Heroicons v2 uses react-icons/hi2)

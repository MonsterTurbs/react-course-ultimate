1) What you are building

We want to replace two hard-coded buttons (Previous and Next) with a single reusable <Button /> component.

  • Same styling and behavior (colors + click handler)
  • Flexible content (text, emoji, ordering of emoji, even multiple elements)

üß† Key idea

Instead of passing content (emoji + label) as separate props, we will pass JSX directly inside the component and read it using props.children.

‚úÖ children is a special prop React gives to every component. It contains whatever you place between the opening and closing tags of that component.

Inside Button, that content becomes available as:

‚úÖ Why it matters

The component becomes truly reusable because it does not need to know its content ahead of time. You just ‚Äúfill the hole‚Äù with any JSX you want.

5) Common mistakes (and fixes) ‚ö†Ô∏è

‚ùå Mistake: Using a self-closing tag

If you write <Button />, there are no children to pass.

Fix: Use opening + closing tags if you want children:

<Button>Click me</Button>
‚ùå Mistake: Forgetting to render children

If your component never uses {children}, nothing shows up.

Fix: Render it inside your JSX:

return <button>{children}</button>;
‚ùå Mistake: Calling the click handler

Don‚Äôt do onClick={handleNext()} (calls immediately).

Fix: pass a function reference:

onClick={handleNext}
‚ùå Mistake: Too many props for content

If you keep adding props for content layout, switch to children.

Fix: pass JSX as children and keep props for behavior/styling.

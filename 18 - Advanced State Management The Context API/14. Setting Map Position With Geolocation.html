1) Create a hooks folder + useGeolocation.js

Why? Custom hooks are reusable. You write it once and drop it into any project later.

Change requested in the lecture: allow a default position and use null instead of an empty object when there is no position yet.

// src/hooks/useGeolocation.js
import { useState } from "react";

export function useGeolocation(defaultPosition = null) {
  const [isLoading, setIsLoading] = useState(false);
  const [position, setPosition] = useState(defaultPosition);
  const [error, setError] = useState(null);

  function getPosition() {
    if (!navigator.geolocation)
      return setError("Your browser does not support geolocation");

    setIsLoading(true);
    navigator.geolocation.getCurrentPosition(
      (pos) => {
        setPosition({
          lat: pos.coords.latitude,
          lng: pos.coords.longitude,
        });
        setError(null);
        setIsLoading(false);
      },
      (err) => {
        setError(err.message);
        setIsLoading(false);
      }
    );
  }

  return { isLoading, position, error, getPosition };
}

4) Sync geolocation position ‚Üí map position

This is the important part: the map only moves when mapPosition changes.

We run an effect whenever geolocationPosition changes, and if it exists, we update the map position.

import { useEffect } from "react";

// ...inside Map component
useEffect(function () {
  if (!geolocationPosition) return;

  setMapPosition([
    geolocationPosition.lat,
    geolocationPosition.lng,
  ]);
}, [geolocationPosition]);
‚ö†Ô∏è Note about effects

This effect introduces an additional render cycle (hook state updates, then map state updates). It‚Äôs acceptable here, but in general React encourages minimizing unnecessary effects.

üß† Mini reviewer (common questions)

Q: Why not set mapPosition directly inside the geolocation hook?

Because the hook should be reusable and not tied to a specific component‚Äôs state. Instead, the component consumes the hook and decides what to do with the position.

Q: Why is the initial position null instead of {}?

null clearly means ‚Äúno value yet.‚Äù An empty object often leads to confusing checks and accidental property access errors.

Q: What‚Äôs the real purpose of the effect here?

üîÅ Synchronization. You‚Äôre syncing two separate systems of state: the hook‚Äôs geolocationPosition and the map‚Äôs mapPosition.

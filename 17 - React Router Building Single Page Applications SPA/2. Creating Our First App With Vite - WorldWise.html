2) üß© SPA + routing: why the URL changes without full page reload

üìå Key term Single Page Application (SPA): the browser stays on one HTML page, but the UI changes as routes (URLs) change.

üîé What you observed in the demo
  • Click Pricing ‚Üí URL changes to something like /pricing.
  • Click Product ‚Üí URL changes to /product.
  • Click Login ‚Üí URL changes to /login.
  • Click the logo ‚Üí returns to the home route /.
  • Inside the app, navigating between Cities and Countries also changes the URL.

‚úÖ This ‚ÄúURL ‚Üí view‚Äù mapping is exactly what React Router will manage in this section.

üß† Beginner mental model

Think of routing as a ‚Äúswitchboard‚Äù: when the path is /pricing, show the Pricing UI; when the path is /app/cities, show the Cities UI, etc. No full page reload required.

3) ‚ö° Why Vite: what changes vs Create React App

‚ö° Vite A modern build tool that offers templates for React and other frameworks.

üÜö CRA vs Vite (in this lecture)
  • CRA: usually comes with many tools pre-configured (e.g., ESLint).
  • Vite: faster dev experience, but you often configure some tooling yourself (e.g., ESLint).
  • Vite uses different scripts: npm run dev instead of npm start.
üéØ Practical benefit you will feel
  • üì¶ Dependency install and dev startup typically feel faster.
  • üß∞ Cleaner entry file compared to CRA‚Äôs default setup.

Course instruction: use Vite v4 in this course so everyone is aligned.

5) üóÇÔ∏è Vite project structure: what‚Äôs different

üìå Notable differences you saw
  • index.html is at the project root (not inside public).
  • Entry file is src/main.jsx (instead of CRA‚Äôs src/index.js).
  • Components commonly use .jsx extension (Vite expects JSX in JSX files).
üß† What to remember
  • React doesn‚Äôt ‚Äúcare‚Äù about your folder structure‚Äîyou do.
  • React‚Äôs job is to render components; the bundler/dev server handles the file setup.

7) üßØ ESLint in Vite: install + configure (so you don‚Äôt code half-blind)

üß† Why it matters ESLint catches unused variables, missing dependencies, bad patterns, and many bugs early.

üì¶ Install the required dev dependencies

8) üßæ Common mistakes checklist

  • ‚ö†Ô∏è Running npm start in Vite (use npm run dev).
  • ‚ö†Ô∏è Forgetting npm install after creating the project.
  • ‚ö†Ô∏è ESLint install command missing install.
  • ‚ö†Ô∏è Creating .eslintrc.json but not adding the Vite plugin.
  • ‚ö†Ô∏è Leaving starter CSS/markup and wondering why UI looks ‚Äúweird‚Äù.

9) üß† Self-check questions

  1. 1) What makes WorldWise a Single Page Application (SPA)? The URL changes (routes) and the view updates without full page reloads.
  2. 2) What command runs the dev server in Vite? npm run dev
  3. 3) What‚Äôs the Vite entry file called (instead of index.js)? src/main.jsx
  4. 4) Why add ESLint? To catch bugs early (unused vars, incorrect patterns) and improve consistency.
  5. 5) What does vite-plugin-eslint do? It integrates ESLint checks into the Vite dev process so warnings show in terminal/overlay.

Key mental model üß†

Network requests take time. JavaScript continues running other lines of code while the request is happening. When the data finally arrives, we run a callback (code) that we registered earlier.

üß© Quick background: Sync vs Async

Synchronous (sync) ‚úÖ

  • One line runs after the other, in order.
  • Example: math operations, string operations, most normal functions.

Asynchronous (async) ‚è≥

  • Some work happens ‚Äúin the background‚Äù (like fetching data).
  • JavaScript does not block (it does not pause the whole app).
  • When the work finishes, JavaScript runs a callback you provided.

In the lecture, the goal is: fetch data from an API endpoint (example: a tutorial API that returns ‚Äútodos‚Äù).

üì¶ The fetch() API returns a Promise

When you call fetch(url):

  • It immediately returns a Promise (üì® ‚ÄúI promise I will give you a result later‚Äù).
  • At first the Promise is usually pending (‚è≥ still loading).
  • Later it becomes either fulfilled (‚úÖ success) or rejected (‚ùå error).

Important note (Quokka / Node) ‚ö†Ô∏è

To use fetch() directly in Node-based tools (like Quokka), you typically need Node.js 18+ because that is when fetch became available in Node by default.

Example: Seeing that JavaScript continues

// 2) This runs right away (JavaScript does NOT wait)

In the lecture, the instructor shows that a normal console.log() prints first, and only later the fetched data prints.

üîÅ Handling the result with .then() (Promise chaining)

To ‚Äúdo something‚Äù when the Promise is fulfilled (‚úÖ), attach a .then() callback.

Step-by-step Promise flow

  • 1Ô∏è‚É£ fetch() returns a Promise of a Response object
  • 2Ô∏è‚É£ You convert the response body to JSON using res.json()
  • 3Ô∏è‚É£ res.json() itself returns another Promise

    console.log("Here is the data ‚úÖ", data);

Why 2 .then() calls? ü§î

  • The first .then receives the Response.
  • res.json() is asynchronous too, so it returns a Promise.
  • The second .then receives the parsed JavaScript data (array/object).

‚ùå Handling errors (rejected Promises)

If something goes wrong (e.g., network error), the Promise can be rejected. A common pattern is to add .catch().

fetch("API_ENDPOINT")
  .then((res) => res.json())
  .then((data) => {
    console.log("Success ‚úÖ", data);
  })
  .catch((err) => {
    console.error("Something went wrong ‚ùå", err);
  });

React connection üß†

In React, instead of only logging data, you typically store it in component state (e.g., setTodos(data)) so the UI updates automatically.

üß™ Mini practice (quick self-check)

  1. Q: What does fetch() return immediately?
    A: A Promise (usually pending ‚è≥).
  2. Q: Why does console.log("Runs immediately") print before the fetched data?
    A: Because JavaScript continues running; it doesn‚Äôt wait for network requests.
  3. Q: Why do we call res.json()?
    A: To convert the response body (JSON text) into a JavaScript value (object/array).
  4. Q: Why might we need another .then() after res.json()?
    A: Because res.json() returns a Promise too.

üìù Quick summary

  • ‚úÖ fetch() starts an HTTP request and returns a Promise immediately.

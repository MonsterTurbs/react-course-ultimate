‚úÖ Big Idea

Tailwind is usually written directly in your markup (JSX/HTML) using many small utility classes.

But sometimes you repeat the exact same long list of utilities (especially for inputs). In those cases, Tailwind provides the @apply directive so you can bundle utilities into a single custom class.

Result: you write a class like .input once, and reuse it everywhere. üôÇ

üß† What is ‚Äúutility-first‚Äù and why @apply exists?

  • Utility-first means: lots of tiny classes (each does one job) + compose them in JSX.
  • @apply is an escape hatch: when a utility combo is repeated a lot, you can compose it into a single reusable class.

‚ö†Ô∏è Important mindset

@apply should be the exception, not the rule. If you overuse it, you slide back into traditional CSS (‚Äúinvent class names for everything‚Äù), which removes many of Tailwind‚Äôs advantages.

üóÇÔ∏è Where do we put these reusable classes?

In the lecture, the reusable class is placed in index.css using Tailwind layers.

Tailwind typically has these directives at the top of your CSS:

@tailwind base;
@tailwind components;
@tailwind utilities;

Then you add a custom layer for your reusable classes:

@layer components {
  /* Your reusable classes go here */
}

Think of @layer components as ‚Äúa safe place to define component-like classes that Tailwind will keep in the right order.‚Äù

üß™ Step-by-step: Creating an .input class with @apply

1) Start with the long utility list (in JSX)

Example: your input has many utilities (rounded corners, padding, border, focus ring, transitions, responsive padding, etc.).

<input
  className="rounded-full border border-stone-200 px-4 py-2
             text-sm transition-all duration-300
             placeholder:text-stone-400
             focus:outline-none focus:ring focus:ring-yellow-400"
/>

When you repeat this across multiple inputs, your JSX becomes noisy and harder to maintain.

2) Move the utilities into CSS using @apply

In index.css (or a global CSS file used by Tailwind):

@layer components {
  .input {
    @apply rounded-full border border-stone-200 px-4 py-2
           text-sm transition-all duration-300
           placeholder:text-stone-400
           focus:outline-none focus:ring focus:ring-yellow-400;
  }
}

Now .input is ‚Äúcomposed‚Äù from Tailwind utilities.

3) Replace repeated utility strings with the reusable class

<input className="input" />
<input className="input" />
<input className="input" />

‚úÖ Your UI looks the same, but your JSX is cleaner.

üß≠ When should you use @apply?

  • Good fit: repeated, long, consistent styles (e.g., inputs, badges, small reusable patterns).
  • Not ideal: using @apply for almost everything (you‚Äôll reinvent ‚ÄúCSS class naming‚Äù problems).

‚≠ê Better long-term reuse in React

For big UI pieces (especially buttons), the cleaner approach is usually to create a React component (e.g., <Button />) and encapsulate your Tailwind class list there.

That way you reuse behavior + markup + styles together, not just a CSS class.

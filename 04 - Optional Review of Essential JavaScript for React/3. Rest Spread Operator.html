üß† The big idea: Rest vs Spread

Both use the same syntax: three dots .... The meaning depends on the context.

Operator What it does How to remember Typical place
Rest ... Collects remaining values into an array/object "Rest" = the rest of the items üß∫ Destructuring (left side)
Spread ... Expands values from an array/object "Spread" = spread items out ü´≥ Creating arrays/objects (right side)

One-liner ‚úÖ

Rest gathers. Spread scatters.

üß∫ Rest operator in array destructuring

In the previous lecture, we destructured the first two genres from an array:

Now if we also want all remaining genres in another array, we use rest:

console.log(otherGenres);

The rest element must be the last element in the destructuring pattern.

If you remove secondaryGenre, then otherGenres would include it too‚Äîbecause ‚Äúrest‚Äù means everything not already taken.

ü´≥ Spread operator with arrays

Use spread when you want to create a new array that contains all elements of an existing array, plus extras.

‚ùå The wrong approach (nested array) üôÖ

‚úÖ The correct approach (flat list) ‚úÖ

Also valid: add to the beginning üîº

Mental model üß†

...genres behaves like you typed each element one-by-one into the new array.

üß± Spread operator with objects

With objects, spread is even more important because it lets you create a new object based on an existing object and:

  • ‚ûï add new properties
  • ‚ôªÔ∏è update/override existing properties

‚ùå The wrong approach (object inside object) üôÖ

  book,
  moviePublicationDate: "2001-12-19",
};

‚úÖ Add a new property correctly ‚úÖ

  ...book,
  moviePublicationDate: "2001-12-19",
};

Reminder üëÄ

Spread expands the existing object‚Äôs properties into the new object.

‚ôªÔ∏è Overriding (updating) properties correctly

To update a property, you typically spread first and then override after. The last property wins. 

// ‚úÖ Override pages
const updatedBook = {
  ...book,
  pages: 1210,
};

Order matters ‚ö†Ô∏è

If you put pages: 1210 first, then spreading ...book after will override it back to the original value. 

// ‚ùå Wrong order (original book overrides your update)

  pages: 1210,
  ...book,
};

Same rule for arrays: build a new array by spreading first, then adding/removing items as needed.

‚öõÔ∏è Why this matters in React (immutability)

In React, you usually treat state as immutable. That means: you don‚Äôt mutate (change) the original array/object. You create a new one.

Why? ‚úÖ

React often relies on reference changes (new array/object) to detect updates. Spread is a standard way to produce those new values.

Typical React patterns üß©

  ...prev,
  role: "admin",
}));

// Update array state (add an item)

You‚Äôll use these patterns constantly once you start managing state.

üéØ Goal of the Component

We have a Steps UI with buttons Previous and Next. The UI shows:

  • Which step is active (1, 2, 3)
  • A message that changes per step

Target behavior: clicking Next increments the step, clicking Previous decrements the step. No manual editing of a variable.

Key idea: to make the UI update automatically, we need state (component memory). üß†

üß© The 3-Step Workflow of State in React

  1. Create a state variable (using useState)
  2. Use that state in your code/JSX (render UI based on it)
  3. Update state inside an event handler (so the UI changes when the user interacts)

‚úÖ If you can do these 3 steps, you can build interactive React components.

1Ô∏è‚É£ Step 1: Create State with useState

useState is a React hook that returns an array with two items:

  • Current state value (example: step)
  • Setter function to update it (example: setStep)

‚úÖ Typical pattern (destructuring)

import { useState } from "react";

function App() {
  const [step, setStep] = useState(1);

  return <div>Step: {step}</div>;
}

Beginner note: The argument to useState(1) is the default/initial value. Here, the step starts at 1.

üîç What useState returns (conceptually)

// Conceptual output
const result = useState(1);
// result is like: [1, function setState() { ... }]

You usually do not store this as result; you destructure immediately.

2Ô∏è‚É£ Step 2: Use State in JSX

Once you have step, you can render UI based on it:

  • Display the number
  • Pick a message from an array
  • Conditionally apply classes (e.g., active step)

Example: render message based on step

const messages = [
  "Learn React ‚öõÔ∏è",
  "Apply to jobs üíº",
  "Invest your new income üìà"
];

function App() {
  const [step, setStep] = useState(1);

  return (
    <p className="message">
      Step {step}: {messages[step - 1]}
    </p>
  );
}

Why step - 1? Arrays are zero-based (index starts at 0). Step 1 maps to index 0.

3Ô∏è‚É£ Step 3: Update State in Event Handlers

To update state, call the setter function (here: setStep) inside an event handler like onClick.

Next button

function handleNext() {
  setStep(step + 1);
}

<button onClick={handleNext}>Next</button>

Previous button

function handlePrevious() {
  setStep(step - 1);
}

<button onClick={handlePrevious}>Previous</button>

Magic moment: Updating state triggers React to re-render the component, showing a new view automatically. ‚ú®

Important: In JSX event props like onClick, pass a function, not a function call. ‚úÖ onClick={handleNext} ‚ùå onClick={handleNext()}

üõ°Ô∏è Preventing Bugs: Bounds/Guards

Without checks, you can go to step 0, -1, or beyond the messages array.

Simple guard clauses

function handleNext() {
  if (step < 3) setStep(step + 1);
}

function handlePrevious() {
  if (step > 1) setStep(step - 1);
}

‚úÖ This keeps step within 1..3 so the UI always makes sense.

Optional improvement (derive max from data)

const maxStep = messages.length;

function handleNext() {
  if (step < maxStep) setStep(step + 1);
}

This avoids hard-coding 3, making the component easier to reuse later.

ü™ù Rules of Hooks (useState included)

Hooks must be called at the top level of a React component function.

Not allowed: calling useState inside if, loops, or nested functions. React (and ESLint) will warn/error. üö´

‚ùå Wrong

function App() {
  if (true) {
    const [step, setStep] = useState(1); // ‚ùå don't do this
  }
  return null;
}

‚úÖ Correct

function App() {
  const [step, setStep] = useState(1); // ‚úÖ top level
  return <div>Step: {step}</div>;
}

Rule of thumb: declare hooks near the top of the component, before handlers and JSX.

‚úÖ Best Practices + Common Mistakes

‚úÖ Update state using the setter only

Use setStep(...). Avoid trying to modify the state variable directly.

// ‚úÖ correct
setStep(step + 1);

// ‚ùå wrong (doesn't update UI properly)
step = step + 1;

‚ö†Ô∏è When you see a value update but UI doesn‚Äôt‚Ä¶

  • Check if you updated state with the setter.
  • Check if your handler is being called (temporary console.log helps).
  • Check bounds (messages[step - 1] can go undefined if step is out of range).

üß† Mental model

  • State = component memory over time
  • Setter = request a new render with updated data
  • JSX = ‚ÄúUI is a function of state‚Äù

üôÇ React lets you avoid manual DOM manipulation. You change state; React updates the UI.

Mini exercise 1: Disable buttons at edges

Add disabled to the buttons:

<button disabled={step === messages.length} onClick={handleNext}>Next</button>

Bonus: style disabled buttons in CSS so they look inactive.

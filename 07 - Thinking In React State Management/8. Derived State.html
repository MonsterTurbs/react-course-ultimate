Derived state means computing values from existing state or props instead of storing duplicate state. This avoids bugs (out-of-sync data) and avoids unnecessary re-renders.

1) What ‚ÄúDerived State‚Äù Means

Derived state is a value you can compute from other data you already have (state or props). Instead of creating a new useState for it, you compute it with normal JavaScript.

  • ‚úÖ Good: compute numItems from cart
  • ‚ùå Bad: store cart, numItems, and totalPrice as separate state and try to keep them synchronized

Key idea: prefer one source of truth. If cart already contains all the data, make cart the source of truth and derive everything else from it.

2) Why Duplicate State is Problematic üöß

  1. Out-of-sync bugs: you must remember to update multiple states together.
  2. Unnecessary re-renders: updating multiple state variables can trigger multiple renders (or at least extra work).
  3. More code, more mental load: extra handlers, extra edge cases.

Beginner mistake: creating two or three state variables when you only need one. If one value can be computed from another, it usually should not be state.

3) Cart Example: Bad vs Good

‚ùå BAD (Duplicate State)

Here, numItems and totalPrice depend entirely on cart, but they are still stored as separate state. 

function Checkout() {
  const [cart, setCart] = useState([]);
  const [numItems, setNumItems] = useState(0);
  const [totalPrice, setTotalPrice] = useState(0);

  function addToCart(newItem) {
    setCart((c) => [...c, newItem]);

    // Now you ALSO have to keep these in sync...
    setNumItems((n) => n + 1);
    setTotalPrice((t) => t + newItem.price);
  }

  return (
    <div>
      <p>Items: {numItems}</p>
      <p>Total: {totalPrice}</p>
    </div>
  );
}
‚úÖ GOOD (Derived State)

Only cart is state. Everything else is computed. 

function Checkout() {
  const [cart, setCart] = useState([]);

  // Derived values (regular variables)
  const numItems = cart.length;
  const totalPrice = cart.reduce((sum, item) => sum + item.price, 0);

  function addToCart(newItem) {
    setCart((c) => [...c, newItem]);
  }

  return (
    <div>
      <p>Items: {numItems}</p>
      <p>Total: {totalPrice}</p>
    </div>
  );
}

Why does this work? Updating cart causes a re-render. On each render, the component function runs again, so numItems and totalPrice are recalculated automatically.

4) Rules of Thumb ‚úÖ

  • Use useState when the value is independent and changes over time.
  • If the value can be computed from existing state/props, derive it instead.
  • Derived state should be a const variable (not a state variable).
  • Keep a single source of truth for related data.

Mental model: ‚ÄúDon‚Äôt store what you can calculate.‚Äù üßÆ

1) Goal of this lecture

We want the user to pick a sorting rule using a dropdown:

  • ‚úÖ Packed status (packed items move to the end)

üìç Placement in JSX:

4) Sorting with derived state üßÆ

Sorting can be calculated from existing state (items) + current UI state (sortBy), so we use a normal variable instead of useState.

‚ö†Ô∏è Critical rule: Do not mutate state

JavaScript sort() mutates the array. If you do items.sort(...), you mutate React state directly (bug + unpredictable UI). Fix: copy first using slice() (or spread) then sort. 

// Derived state (no useState needed)
let sortedItems;

if (sortBy === "input") {
  sortedItems = items;
}

if (sortBy === "description") {
  sortedItems = items
    .slice()
    .sort((a, b) => a.description.localeCompare(b.description));
}

if (sortBy === "packed") {
  sortedItems = items
    .slice()
    .sort((a, b) => Number(a.packed) - Number(b.packed));
}

Why these sorting techniques work

  • üî§ Alphabetical: localeCompare handles string comparison properly (including locale rules).
  • ‚úÖ Boolean sort: Number(false)=0, Number(true)=1, so sorting by 0 - 1 moves packed items down.
‚úÖ React-friendly mental model:

State (items) is the ‚Äúsource of truth.‚Äù Sorting is just a ‚Äúview‚Äù computed from it. When items or sortBy changes, the component re-renders and sortedItems is recalculated automatically.

6) Common mistakes (and fixes) üßØ

‚ùå Mistake: Using state for sortedItems

Creating const [sortedItems, setSortedItems] = useState(...) adds sync problems.

‚úÖ Fix: Use derived state (a normal variable) computed from items + sortBy.

‚ùå Mistake: Mutating items with sort()

items.sort(...) mutates the array (state mutation).

‚úÖ Fix: items.slice().sort(...) or [...items].sort(...).

‚ùå Mistake: Missing onChange

If <select value={sortBy}> has no onChange, it becomes read-only.

‚úÖ Fix: onChange={(e) => setSortBy(e.target.value)}.

‚ùå Mistake: Incorrect option values

If option values don‚Äôt match your conditions, sorting won‚Äôt change.

‚úÖ Fix: Ensure value="input", "description", "packed" align with your if checks.

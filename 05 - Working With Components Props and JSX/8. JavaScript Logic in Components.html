üß© Big idea: components are just functions

A React component is a JavaScript function. That means you can run any JavaScript inside it. The code runs when the component function is executed (rendered).

Remember ‚úÖ

  • üß† The function body runs when the component renders.
  • üßæ The return (...) returns JSX, which describes the UI.
  • üîÅ If the component re-renders, the function body runs again.

üìç Where logic can live

So far you used small JS snippets inside JSX (like {new Date().toLocaleTimeString()}). But you can also write logic above the returned JSX.

Inside JSX (small expressions) ‚úÖ

return (
  <footer>
    We're currently open {new Date().toLocaleTimeString()}.
  </footer>
);

Good for short expressions.

Outside JSX (compute first) ‚úÖ

const hour = new Date().getHours();
const message = hour >= 12 ? "Open" : "Closed";

return <footer>{message}</footer>;

Better for multi-step logic and readability.

üïí Example: get the current hour

You can create variables inside the component and log them to the console.

üö´ Why alert() is a bad demo in real apps

The lecture uses alert() just to prove that JS runs inside components. But alert() is blocking (it pauses JavaScript execution).

In real apps ‚úÖ

  • Prefer rendering a message in JSX instead of blocking alerts.
  • Or show a non-blocking UI notification (toast) later using libraries.

üîÅ Strict Mode: why you saw things run twice

You may see the alert appear twice in development. The lecture explains this is due to Strict Mode, where components can render twice to help detect side effects.

What to remember ‚úÖ

  • üß™ In development, React may invoke render logic more than once.
  • ‚ö†Ô∏è Side effects like alert() become very noticeable.
  • ‚úÖ This is one reason to avoid side effects directly in render logic.

‚úÖ Better pattern: compute isOpen boolean

Instead of triggering alerts, compute a boolean and then use it for UI decisions. This becomes useful later when you conditionally render UI. 

function Footer() {
  const hour = new Date().getHours();
  const openHour = 12;
  const closeHour = 22;

  const isOpen = hour >= openHour && hour <= closeHour;
  console.log(isOpen);

  return (
    <footer>
      We're currently {isOpen ? "open ‚úÖ" : "closed ‚ùå"}
    </footer>
  );
}

Key takeaway üîë

Put multi-step logic above the return, compute values like isOpen, then let JSX display the result.

1) üì¶ Install the bridge library

In your project folder:

(Redux by itself is framework-agnostic. React Redux is the React-specific connector.)

2) üß© Provide the store to the React app

Just like Context, we wrap the app with a Provider. The Provider receives a single prop: store. Once wrapped, any component can read from the store.

‚úÖ Example: index.js

// index.js

import ReactDOM from "react-dom/client";

import App from "./App";
import store from "./store";

import { Provider } from "react-redux";

const root = ReactDOM.createRoot(document.getElementById("root"));
root.render(
  <React.StrictMode>
    <Provider store={store}>
      <App />
    </Provider>
  </React.StrictMode>
);
üí° Why this works: Provider places the store into a React context internally, and then React Redux hooks (like useSelector) can access it anywhere below.

3) üìñ Read state from the store with useSelector

useSelector takes a callback function. That callback receives the entire Redux state tree. Your job is to select the exact data your component needs.

‚úÖ Example: read customer name

  // Selector: pick ONLY what this component needs
  const fullName = useSelector((store) => store.customer.fullName);

  return (
    <h2>üëã Welcome, {fullName || "Guest"}!</h2>

üß† What ‚Äúsubscription‚Äù means

When a component uses useSelector, it becomes subscribed to the store. Whenever the store updates, React Redux checks whether the selected value changed.

  • ‚úÖ If the selected value changed ‚Üí component re-renders.
  • ‚úÖ If it did not change ‚Üí component does not need to re-render.
‚úÖ Practical tip: Select the smallest data you need. For example, select store.customer.fullName, not the entire store.customer object, unless you truly need the full object.

üö´ Common mistakes (and fixes)

  • Problem: useSelector throws an error about missing store.
    Fix: Ensure the component is rendered inside <Provider store={store}>.
  • Problem: store.customer is undefined.
    Fix: Verify your root reducer keys match your selector path.
  • Problem: Too many re-renders.
    Fix: Select smaller pieces of state (primitives when possible).

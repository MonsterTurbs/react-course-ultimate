üß† Core idea: ‚Äúmultiple returns‚Äù (early return)

Until now, your components usually had a single return. But React components are just JavaScript functions, so you can use multiple return statements‚Äîas long as only one is executed.

Important ‚úÖ

  • A component still returns one JSX result per render.
  • But which JSX is returned can depend on a condition (early return).

‚öôÔ∏è Why this works

Early returns happen outside JSX, directly inside the component function body. That means you can use normal JavaScript statements like if‚Äîunlike inside JSX curly braces.

Remember üß©

Inside JSX { } you need expressions. Outside JSX (in the function body), you can use full JS: if, return, variables, etc.

function Component() {
  // ‚úÖ normal JavaScript here
  if (someCondition) return <p>A</p>;

  // default return
  return <p>B</p>;
}

Lecture takeaway üß†

Early return is often best when you render an entirely different component (or nothing). If you just need to swap a small piece, ternary or && can be cleaner.

üß≠ When to use early return vs && vs ternary

Use && when‚Ä¶ ‚úÖ

  • You want ‚Äúrender this or nothing‚Äù
  • Your condition is clearly boolean (avoid 0 showing up)

Use ternary when‚Ä¶ ‚úÖ

  • You want ‚Äúrender A else render B‚Äù
  • You want explicit empty state / alternative UI

Use multiple returns when‚Ä¶ ‚úÖ

  • You want to exit early (guard clauses)
  • You want to return a totally different component
  • You want to render nothing with return null;

return <Pizza />;

Practical recommendation üß†

Early returns are great for ‚Äúguard clauses‚Äù (loading/error/no access/sold out). For small inline UI swaps, ternary or && is usually more readable.

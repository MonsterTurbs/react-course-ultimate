üß© Step 3: useForm, register, handleSubmit

In the form component (CreateCabinForm), call useForm(). It returns helpful functions.

üß∑ register

Connects an input to React Hook Form so RHF can read its value.

üßæ handleSubmit

Wraps your submit handler. It gathers values from registered inputs and then calls your function with the final data.

import { useForm } from "react-hook-form";

export default function CreateCabinForm() {
  const { register, handleSubmit } = useForm();

  // later: more helpers like formState.errors, reset, etc.

  function onSubmit(data) {
    console.log(data);
  }

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      {/* inputs will be registered here */}
    </form>
  );
}

üß† Beginner tip: Think of handleSubmit as a ‚Äúcollector + validator + caller.‚Äù It collects field values and then calls your onSubmit with a clean object.

üìù Step 4: Register inputs (no controlled inputs needed)

Notice the form inputs are not controlled (no useState per field). React Hook Form works great with uncontrolled inputs.

‚úÖ Result: no manual onChange handlers and no state variables per input. RHF injects the needed props for you.

How registering looks

You ‚Äúspread‚Äù the output of register("fieldName") into the input. This looks unusual at first, but it‚Äôs standard RHF usage. 

// Example inputs (simplified)

<input {...register("name")} />
<input type="number" {...register("maxCapacity")} />
<input type="number" {...register("regularPrice")} />
<input type="number" defaultValue={0} {...register("discount")} />
<textarea {...register("description")} />

// File input is often handled a bit differently ‚Äî we'll do it later.

What does register() actually do?

RHF adds props like onChange, onBlur, and ref to the input. You can see these in React DevTools when you inspect the element.

üîé DevTools observation: after registering an input, you‚Äôll see extra props/functions on that input. That‚Äôs RHF wiring itself into your form.

üöÄ Step 5: Submit handler and captured data

The submit flow is:

  1. User clicks the submit button (inside the form).
  2. Form triggers onSubmit event.
  3. RHF runs handleSubmit(onSubmit).
  4. RHF collects values from all registered fields.
  5. RHF calls onSubmit(data) with a data object.
function onSubmit(data) {
  // data is an object with all registered fields
  // Example output:
  // {
  //   name: "Test Cabin",
  //   maxCapacity: "5",
  //   regularPrice: "500",
  //   discount: "100",
  //   description: "..."
  // }
  console.log(data);
}

<form onSubmit={handleSubmit(onSubmit)}>
  ...
</form>

‚ö†Ô∏è Note about types: HTML inputs usually give you strings (even for numbers). Later you can validate/transform as needed (RHF supports value parsing options).

At this stage, the goal is simply to prove: we can submit and receive all form values without manual state handling.

üìö Quick review + mini quiz

Cheat sheet üß†

  • const { register, handleSubmit } = useForm()
  • {...register("fieldName")} on each input you want RHF to manage
  • onSubmit={handleSubmit(onSubmit)} on the <form>
  • onSubmit(data) receives all registered values
